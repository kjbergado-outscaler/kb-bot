Orders Section


By clicking on the order section you will have access to the main order list in chronological set (last created order first), where you can easily navigate through.






By clicking on a specific order reference number, you will be redirected to the Order Details Page. where you can find all information related to the order.



Order Details Page





Summary / Order information


The Summary card displays a brief summary of the order. This summary includes the following information.

  • Order Type:The type of order. An Order can be either of the following types:
  • HD for Home Delivery Orders.
  • CC for Click and Collect Orders.


  • Order Status: The current status of the order. The standard workflow contains the following standard statuses: Pending, Booked, Processing, Shipped, Complete, Canceled, or Returned. More details in the article: OMS Concepts | definitions and logics



Customer Information


The Customer info card displays information related to the customer that placed the order.

  • Customer Name: This is the name of the customer that has placed the order.
  • Phone Number: This is the phone number of the customer that has placed the order.
  • Email: This is the email of the customer that has placed the order.
  • Ref: is the customer ID (future reference number of customer account)



Retailer (Market store information)


The Retailer info card displays information related to the retailer (Market).


  • Retailer ID: This is a unique ID that is associated with the retailer that the order has been placed.
  • Retailer: This represents the Country or Market where the order has been placed against.





Delivery Information (customer)


The Delivery info card displays the delivery information of the order.
  • Pickup Location Ref: This is a reference number that is associated with the store that the order will be picked up from. The 'Pickup Location Ref' is only applicable for Click and Collect orders and will be left blank for Home Delivery orders.
  • Customer Address: This can be either the store address that the order will be picked up from or the customer's address that the order will be delivered to.




Order Items


The Order Items card displays the following information:

  • Ref: The unique reference number related to the item ordered (SKUs).
  • Product Name: The name of the item ordered
  • Price: The price of the item ordered

By default, the Order Item card displays up to 10 items per page. To view more than 10 items, users need to click on the number and select either 25, 50 or 100 items to be displayed.



ADDITIONAL INFORMATION - STANDALONE X NON-STANDALONE ITEMS:

A standalone item is how we refer to a product that has only one SKU and also appears in OMS with only one SKU associated.


The non-standalone items will be the ones that have only one SKU, however, due to system issues it will appear in FluentCommerce in the same way as a bundle or composite. But you check it, it is indeed only one item.

Email Sent


The Email Sent card displays the following information:

  • Email Category: The type of email sent to the customer.
    • SendOrderConfirmationEmail - email sent once the order status is up to Booked.
    • SendShippingEmail - email sent once the order status is up to Order Shipped.
    • SendOutForDeliveryEmail - email sent once the order status is up to Order Shipped and the order is on the way to the customer's house.
    • SendOrderDeliveredEmail - email sent once the order status is up to Completed.
    • SendInvoiceAsEmail - email sent once the order status is up to Completed.
  • Customer Email: This is the email of the customer that has placed the order.
  • Date & Time: This is the date & time that the customer has placed the order.
  • Delivery Status: Status of the email (Sent).

Related Tab


Related Tab is the section of the order that centralizes any supplement or return initiated to the specific order.





Returns details:

  • Return Order reference: Unique platform-generated ID associated with the return.
  • Status: Current status of the return. Possible values are:
    • Created
    • Booked
    • Pending Refund (to be validated if it will be included in production)
    • Completed
  • Created: The date the return was created.



Comments Tab



This tab is where all comments tagged to the order will be shown.


If a customer contacts CS and has a request, or if there is any specific information that should be shared related to the order, we should ensure we add a comment to the record. By doing that, you can always review and verify all the previous notes related to the order left by other Emmies.




Comments Tab details:

  • Comment: Displays comments updated for the selected order.
  • Created On: Displays the date the comment was created.
  • Updated On: Displays the date the comment was updated.
  • Name: refers to the users' name who posted the comment (Developers are still working on ensuring the NAME OF THE USERS will be added in the comments tab)

Additional Tabs


Transactions Tab


You can find all updated information related to the financial transactions of the order, such as payment, refund, compensation, and discounts. The Transaction REF Column shows a line-per-line update regarding this screenshot below.


This tab displays the payment type, reference number, date, status, and amount of an order.





Transaction Tab details:

  • Ref: This is a unique reference that is associated with the transaction of the selected order. The last part of the ref is regarding its status (i.e. Charged successfully, authorized successfully, refunded successfully, etc.)
  • Created On: This displays the date the transaction was created
  • Status: This displays the transaction status of the order.
  • Type: This displays the type of transaction associated with the order.
  • Card Type: This displays the type of card used for this transaction.
  • Payment Method: This displays the payment method for the transaction.
  • Amount: This displays the amount paid (in $) for a transaction.




Attributes Tab


Attributes are additional tech information that refers to the order, such as payment provider information, taxes ID; etc.



Order Attributes Tab details:

The three attribute fields are as follows:

  • Name: This displays the name of the attribute associated with the order.
  • Type: This displays the type of attribute associated with the order.
  • Value: This displays the value of the attribute.

Fulfilment Section




Fulfillments are the direct equivalents of BARCODES in Magento.


Check status, find fulfillment location information & manage the fulfilments under the fulfillment section (e.g. edit / cancel it)





The other way of viewing fulfillments was already detailed above in the Navigating - Order management: Fulfilments tab



Fulfilment Section details:

  • Fulfilment Ref: This is a unique ID that is associated with the fulfillment
  • Order Ref: the Order number reference where the fulfillment is tagged
  • Customer: Reference of the customer name
  • Fulfilment Status: This displays the fulfillment status of the order
  • Fulfilment Location: This displays the fulfillment location assigned for this fulfilment (warehouse)
  • SKU: what is the SKU related to the fulfilment
  • Retailer name: market related
  • Creation Date: The date when the fulfillment was created

    By default, the fulfillment tab displays up to 10 items per page. To view more than 10 items, users can click on the number and select either 25, 50 or 100 items to be displayed.

Credit Memos Section



The credit memos tab shows a list of all credit memos, which can be filtered accordingly.


A credit memo is a financial term for a monetary transaction, e.g. a refund to the customer.


After an item cancellation, a credit memo is automatically created. You can immediately find it by accessing the Credit memo in the Menu bar and it will be related to the fulfillment ID.





The Credit memo (CM) page contains:

  • Credit Note Ref: the unique reference for the CM
  • Order: Order Reference tagged to the CM
  • Amount: Total Value of the CM
  • Date: Date the CM was created
  • Status: Refers to the CM Status, which is described in the table below (Credit Memo Status)





Credit Memo status

The credit memo will always display the total amount of the transaction & you will be also able to find the status of it, in accordance with our Emma Refunds team update.



StatusWhat does it mean?
Createdcredit memo has been created
Requestedcredit memo has been acknowledge
PSP Pendingcredit memo is processing
Completecredit memo is completed
Failedcredit memo has failed
