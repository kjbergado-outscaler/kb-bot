What is Automatic Ticket Assignment?

For each Freshdesk group, we can configure the distribution and balance the number of tickets assigned to agents within the Group. To enable or modify this for your market please submit a ticket to the IT Portal.

  1. Round Robin ticket assignment - Assign tickets to agents in a circular order (no special logic)

  2. Load balanced ticket assignment - Tickets are assigned based on an agent's load. Configure agent load under Omniroute ™ settings. Omniroute settings can be directly configured by Supervisors.

  3. Skill based ticket assignment - Assign tickets to agents based on their skills (see "What are Skills?" below). When a skill is mapped to two or more agents, incoming tickets will be assigned to the agent with a lesser load. Only IT or CE Global team should create Skills. Please log a ticket if needed.

  4. Manual (not recommended) - automatic ticket assignment is turned off. Either agents pick from the queue or supervisors assign tickets manually.



What is Agent Availability?

Agent Availability determines whether or not the agent is available to be assigned tickets.

To enable or modify this for your market please submit a ticket to the IT Portal.

  1. Allow agents to control their availability status - Agents can choose to go offline for automatic ticket assignment.

  2. Allow only admins/supervisors to control agents’ availability status - Only supervisors can change agents' status using the availability dashboard.



What are Skills?

  • Skills allow you to easily match tickets to the right agents. You can associate distinct skills to each agent and create rules to make sure that only tickets matching those skills are assigned to them.

  • If the skill mapped to a ticket matches an agent's skill, the ticket will be assigned to the agent.

  • The market can choose to rearrange the Skills depending on priority. Some agents can be assigned more than one skill. Whichever skill is first in order would be prioritized for the agents.

  • The load is computed at a group level i.e. even if the agent is a part of multiple groups, the load that you have set under the Skill-based assignment checks only for the open tickets in that group.

  • Sample (from FR group):

    • The Marketplaces skill is first in order so if Agent A is in both Marketplaces and Post Delivery skills, the agent will be assigned Marketplaces tickets first.

  • Skill sample: FR - Pre Delivery

    • Skill conditions must be existing ticket fields (cannot be Tags, Requester email addresses, or words in the Subject).

  • To enable or modify this for your market please submit a ticket to the IT Portal.


Freshdesk Skill Naming Convention

  • Skills should use the following convention:
    • General Skills (main skill for assignment, no specialties) - <Freshdesk Group> - General
    • For one group with specialty: <Freshdesk Group> - <Specialty>
    • For multiple groups: <Group1><Group2> - <Specialty>

How to toggle Agent Availability

  • If you are an agent:

    1. Check the upper right corner of your Freshdesk Tickets page: Helpdesk : Bettzeit GmbH (freshdesk.com). If you are unable to see this icon, this means you are not assigned to a group that is using Automatic Ticket Assig

    2. The moon icon indicates that the agent is Unavailable:

    3. Click on the icon to toggle agent availability as needed. Available will allow the agent to start receiving tickets.

  • If you are a supervisor:

    1. Go to your Freshdesk dashboard: Helpdesk : Bettzeit GmbH (freshdesk.com)

    2. Scroll down to the Available agents section and click on View details:

    3. Filter by your Market:

    4. Look for the agent and click on the dropdown beside the status to toggle:

  • Markets usingShifts - inquiries are automatically assigned only when an agent is available at their shift. Agents are automatically set to Available at the start of their shift, and set to not available upon ending their shift.

    • not all markets use this and shifts are assigned PER AGENT not per GROUP. Refer to the market setup file to check if your market is using them.
