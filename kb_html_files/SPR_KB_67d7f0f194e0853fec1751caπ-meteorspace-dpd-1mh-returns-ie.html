Manually creating returns with MeteorSpace

Step 1: Check for Active Returns: Ensure there is no active return in Fluent Commerce for the same items (the customer might have already created a return via the portal).

Step 2: Create Return in OMS (Fluent):

  • Carrier field: Fluent Commerce: MeteorSpace (If they reply that DPD is going to service the area, use DPD_IE)

  • Pick up date: Leave empty

  • Return status: Do not move to 'Organized'

How to verify if the return was created correctly

  • The customer contact details are correct

  • The return status must be set to 'Created'

  • The pick up date must be left empty (Until confirmed by the Carrier)

  • The carrier field is populated with: MeteorSpace / DPD_IE

Note: The return must never be manually set to 'Organized'. If the return status is set to 'organized' manually, the return will not be recognized by our platforms, and it will never be forwarded to the return carrier. Please ensure the return status is 'Created' after you created the return.

Note: MeteorSpace (Vanex) does not have a tracking link. Please disregard the 'Open Carrier Trace' link on Parcel Lab

How are status updates handled for the return

  • Status updates for Fluent Commerce are not automated.

  • You (the agent) must monitor the return status on the email thread with the carrier on the planned pick-up date.

  • Follow up after one week to ensure the return is processed.

  • Update the return status in the Order Management System (OMS).

How is the Refund Triggered?

Note: This process applies to returns within the trial period

When to Start the Refund Process: The carrier will contact you once the items have been collected from the customer. Start the refund process once the carrier informs you that the product was picked up.

If the customer contacts you but does not have the document, please contact the carrier (Meteorspace) via email to verify if the collection has happened.

To process the refund, you must complete the Order: In the OMS (Fluent Commerce), mark the return as 'complete' to trigger the refund automatically.

Refund timeline: The refund time starts once the order is completed in OMS. Our refunds team (ARD2C) usually releases the refund within 2 working days. After that, it will depend on the customer's own payment provider how long it takes before they see the refund in their bank account.

Note:

  • No Proof of Payment: Do not request proof of payment or product pictures from customers or carriers.

  • Unresolved Tickets: For tickets open for more than 14 days, create a JIRA ticket in "Order issue" at the Customer Excellence Portal .

When can this return ticket be considered 'Resolved'

Once:

  • The customer reported the return was performed

    AND

  • You have updated the status of the return in the OMS to 'COMPLETE'

    AND (in case of refunds)

  • You have informed the customer of the expected timeline for the processing of the refund.

The return can be considered 'resolved' and you can close the case.
