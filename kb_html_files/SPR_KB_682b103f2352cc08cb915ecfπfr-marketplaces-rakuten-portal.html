Platform Overview

The Rakuten interface includes the following key sections:

  • Orders

  • Messages (Inbox)

  • Incidents (Claims)

  • Returns

Use the left-hand navigation bar to access each section.

​

​

Order Management

Viewing Orders

  • Go to Orders > All Orders to see current, shipped, or pending orders.

  • Click an individual order to view:

    • Customer details

    • Product purchased

    • Delivery status

Cancellations

  • Cancellations must be made in Fluent Commerce only.

  • You do not cancel or manage the cancellation in Rakuten.

  • A dedicated Rakuten service team handles refund processing and formal cancellation steps once Fluent is updated.

Returns

  • Returns are also processed entirely in Fluent Commerce.

  • Do not use Rakuten to create or manage return workflows.

  • Ensure that Fluent reflects the correct return status.

Customer Messages

  • All customer communication must be handled within the Rakuten portal.

  • To view messages: go to Inbox or Messages.

  • Read and respond directly inside the platform.

  • Review message count daily and ensure no messages are left unread.

​

​

Claims / Incidents

  • Incidents include missing items, non-delivered packages, product issues, etc.

  • These are listed under “Reclamations.”

  • When a new incident is opened:

    • Review the claim

    • Investigate via Fluent Commerce (e.g., check fulfillment or tracking info)

    • Reply with an appropriate response directly through Rakuten

General Notes & Behavior

  • Do not open or reply to messages if you are not the designated agent, as that removes them from the shared inbox.

  • Do not cancel, refund, or return orders inside Rakuten.

  • If there's a technical issue with login or sync, escalate to the marketplace manager or Francisco via JIRA.
