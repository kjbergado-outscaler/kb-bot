Fluent Status - Work Flow

Scope

This SOP explains the different statuses (every) order(s) go through. The
goals is to understand where an order is and what to do in case it is blocked.

Fluent Status

Fluent Commerce is an order management System run by CommerceTools.

 

When an order is entered in the system, it goes through different status for
the order and and the fulfilment.

An order can be composed of one or more fulfilment.

Fulfillment is a unique ID to track the journey of a request created in a
warehouse, with one or more items, at any point in time.

Every time an order is placed in our system, a logic behind will group all
items according to their warehouse location, and each of these groups will be
assigned a fulfillment ID.

Every order status has multiple status for the fulfilment. The following
status shows the chronological path an order goes through.

Order Booked

The order has been created and the payment has been made. This is the first
status for every order.

The order should not be in this state for longer than a day if there is
stock.

If there are issues in this status you can´t solve, please raise a ticket
here:

Service Request Handling - Jira Service Management

ASSIGNED

·       Description:
The order has been assigned a fulfilment location but has not been transmitted
to it.

·       Issue:
This might be due to a lack of “external_order_number”.

WAITING_FOR_STOCK

·       Description:
One or more items in the order are out of stock. The fulfilment is therefore
marked as WFS and all items in the fulfilment will be marked as WFS even though
there is stock.

·       Issue:
Check the inbound shipment schedule in the next days/weeks and warn Sales and
CS agents about this situation / delays.

PENDING

·       Description:
One or more items in the order are out of stock. The fulfilment is therefore
marked as WFS and all items in the fulfilment will be marked as WFS even though
there is stock. (Pending serves for marketplace orders as “WAITING_FOR_STOCK)

·       Issue:
Check the inbound shipment schedule in the next days/weeks and warn Sales and
CS agents about this situation / delays.

CARRIER_ALLOCATION_PENDING (not
every order goes through this status)

·       Description:
Once the order has been created, a transporter will automatically be assigned
through the software “WOOP”. It can happen that this transmission fail due to a
problem in the address or the name of the customer.

·       Issue:
If a fulfilment remains in this status for more than a day, please check the
possible root causes:

o  Missing or invalid customer address information
(e.g., special characters, misspellings).

o  A system error in the carrier allocation process
and/or problems with the carrier integration (e.g., EDI or configuration
issues).

SCHEDULED (not every order goes
through this status)

·       Description:
If an order is planned to be fulfilled and shipped further in the future it
will stay in this status until it´s time to be prepared. Verify if the order is
transmitted to the warehouse 5 days before the requested delivery date.

·       Issue:
If an order should not be in this status, please raise a ticket.

Order Processing

The order has been received by the warehouse and is now being prepared.

The order should not be in this state for longer than ca. 3 days.

For all fulfilment status, if the order stays too long in this status, an
investigation must be run though the warehouse:

Cancellation
Process: At this stage you cannot easily cancel an order. You first need to
align on with the warehouse. After you received the confirmation that it can be
cancelled, please raise a ticket to Fluent in order to mark this order as
cancelled.

SUBMITTED_TO_FL

·       Description:
The order has been transmitted to the warehouse.

RECEIVED_BY_FL

·       Description:
The warehouse has taken the order into account and will start preparing it.

PICKED_PACKED_BY_FL

·       Description:
The warehouse is preparing the order.

READY_TO_DISPATCH

·       Description:
The order is ready to be shipped.

REJECTION_PENDING (not every
order goes through this status)

·       Description:
The order will be rejected by the warehouse, it will take a little time until
it is in the ‘rejected’ status to then refulfill this order

·       Issue:
The order should not stay in this status and should move on to “rejejected” /
“action required”

 

Order Shipped

The order has been shipped and in is now in the transporter’s hand.

The order should not be in this state for longer than ca. 3 days.

For all fulfilment status, if the order stays too long in this status, an
investigation must be run on the carrier.

There are two types of returns. One where the product arrived at the
customer and is then returned and the other type where the order never arrives
at the customer at the first place.

Return Type 1

These are order statuses.

Order RETURN-CREATED

The customer is not satisfied with its order and is being send back to Emma.
This is a manual return created following a customer request (meaning the
product arrived at the customer in the first place) - Return order

Order RETURN_COMPLETE

The product was never delivered / accepted in the first place. The products
are now back at the warehouse. The return is complete.

Return Type 2

These are fulfilment statuses.

RETURN_SENDER_PROGRESS

·       Description:
The product was never delivered / accepted in the first place. The product is
back on its way to the warehouse.

RETURN_SENDER_COMPLETE

·       Description:
The product was never delivered / accepted in the first place. When the product
is back at the warehouse from the carrier it gets this status. The return is
now complete

Return order

  • Return Reasons

A customer may ask for a return if there is an issue with their order (The
product have been damaged or they received the wrong product….).

Return Reasons

For Retail in France, the return are created and managed by the CS agents.

They receive the customer product or logistic claim through Freshdesk and
ask for photos.

Two cases appear:

·       The
customer wants to receive new products, a new order and the return are created.

·       The
customer ask for reimbursement, the return is created along with a credit memo.
Once the products are received by the warehouse, the reimbursement is sent and
the order is update on Fluent.

NB : The return can be made at the store or at the final customer place
directly.

Two types of returns are possible:

·       Return
following a Product Claim (the product is damaged or the client is unsatisfied
after the 100 nights).

·       Return
following a Logistic Claim (an error in the dimension or the product itself).
