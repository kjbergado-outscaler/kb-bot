What is Agent Capacity?

Agent Capacity in Sprinklr defines how many cases an agent can handle at once across different channels. Each case type is assigned a weight, and agents have a total capacity of 100. The system automatically assigns cases based on this configuration.

The standard Emma configuration is called “Emma Agent Capacity - General”.

Who Can Modify Agent Capacity?

Only the Sprinklr Team and Emma Tech Team are authorized to change agent capacity details. BPO personnel do not have access to these configurations.

If you find an Agent without the Capacity set-up as per the image above, please raise a ticket to the Emma IT Team via the IT JIRA Portal.

Understanding the Capacity Configuration

The total agent capacity is set at 100. And each case type consumes part of this capacity based on its weight.

Main Case Types and Weights:

  • Voice Call – 100

  • Live Chat – 33

  • Social – 25

  • Email – 10

  • Live Chat Deflected – 33

  • Deflected to Email – 10

Overcapacity Situations

Check the scenario in which overcapacity - capacity showing more than 100 - can happen:

Deflection from Email to Chat:

  • If a case starts as an Email (10 weight) and then deflects to Live Chat (33 weight), the agent's total capacity may exceed 100.

  • If the total exceeds 100, the overflow function will not work.

  • If deflection happens to a live channel, overflow also won’t work until the case is removed from the agent’s assignment.

​

Manual or Macro Assignment Over 100:

  • If a case is manually assigned, assigned based on stickiness/preferred assignment (pending confirmation) or assigned via macro, it may push the agent’s capacity beyond 100.

Case Assignment

Automatic Case Assignment

Sprinklr automatically assigns cases to agents based on capacity availability:

  • Agents do not handle voice and chat simultaneously;

  • Once an agent’s capacity reaches 100, no more cases will be automatically assigned;

  • An overflow of one additional chat or voice case is allowed only if the agent's capacity is filled with offline cases (email or social);

  • Cases only count as consumed capacity when assigned to an agent;

  • Macros should be used to update case status and remove assignments when necessary, ensuring agents are not overloaded.

Manual Case Assignment

Cases can be manually assigned to an Agent. However, we don't instruct agents to do it unlesse authorized by the Supervisor.

Those are the two ways to manually assign cases:

  1. Using the "Assign to X" Macro: Only for Supervisors to use. The Supervisor can apply the macro "Assign to X" and select to which agent the case will be assigned to.

  2. Clicking the “Assign to Me” button: Agents can only do it if authorized by the Supervisor.

Checking Assignment Type

To see how a case was assigned:

  1. Open the Queue Monotoring tab.

  2. Double-click on the agent’s “Detailed Consumed Capacity”.

  3. A side panel will open showing all assigned cases.

  4. The last assignment type will be listed at the bottom of each case panel:

    • FAIR_ASSIGNMENT – Assigned via automatic fair routing.

    • PREFERRED_ASSIGNMENT – Assigned based on preferred routing settings.

    • MACRO – Assigned via macro action.

    • MANUAL – Assigned manually by a user.
