Introduction


In this article, you find the instructions and links to use the Returns team JIRA Portal. Please read carefully the specifications about when to open tickets with this team. Any ticket opened with the wrong purpose will not be answered and re-directed to another portal.


When to use the Returns JIRA Portal


The new Return Stuck Escalations portal will cover any return-process-related issues that lead to returns being stuck (Fluent gaps and bugs; and future issues with the returns portal or with the returns widget). Reporting and escalating those cases will also help the Return Team with investigating and creating solutions for issues.


At this moment, the Returns Team is already aware of a recurring bug in FluentCommerce that is making items non-returnable. Any similar cases to this, still can be raised to them so they can fix them manually.



Examples of cases to be escalated


Example 1:


The customer wanted to request a return. The agent went into FluentCommerce and saw that the item show "Returnable Quantity = 0", even though the item was already delivered and there was no previous returns created -> This is a bug. This is an example of a case that should be raised in the Return Escalation Portal.

Example 2:


If a customer wants to return a bundle but FluentCommerce is showing "Returnable Quantity = 0" -> This is a bug. This is an example of a case that should be raised in the Return Escalation Portal.
