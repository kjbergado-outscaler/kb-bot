1. How to initiate a return

Step 1: Ensure customer's phone number, email, and pick-up address are correct.

Step 2: For 2-man-handling returns, select the 2mh pick-up/return service for all items.This ensures everything is picked up together.

Step 3: Set the carrier fields as: set to '4PL CS'

Step 4: In Magento, set Packaging material to 'No'.

Step 5: Leave the pick-up date field empty.

Step 6: Once created, the return status will be 'Created'.

2. Important information for setting customer expectations

Dynalogic will manage all pick-up communications. Customers can expect:

  • An automated email with a link to schedule the pick-up.

  • Email and phone reminders to make the appointment.

  • A reminder on the pick-up day.

  • A final reminder 30 minutes before the pick-up.

3. How to verify if the return was created correctly

For Magento

  • The customer contact details are correct

  • The carrier field is populated with '4PL CS'

How are status updates handled for the return?

  • 4PL CS will automatically update the status in the Order Management System (OMS). It may take 7-10 days to reflect in Magento.

  • For detailed updates, check 4PL's Optilo platform (see 'the article about 4PL Returns' article below).

  • If no updates appear in Magento after 10 days, please escalate the return status update via the 4PL CS Jira portal.

4. How the refund is triggered

  • Returns are completed automatically in Magento once the carrier picks up the items.

  • Refunds are initiated automatically once the return is marked as complete.

  • Banks or payment service providers (PSP) have different timelines for processing refunds. Check the 'Refund request' article for details.

  • If customers have questions after the refund is completed, they should contact their bank or PSP for more information.

5. When can this return ticket be considered 'Resolved'?

Once you have successfully created the return and have verified that it was created correctly, you can mark the case as 'resolved'.
