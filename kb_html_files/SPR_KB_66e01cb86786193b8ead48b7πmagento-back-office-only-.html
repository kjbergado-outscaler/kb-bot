Address Correction


Incorrect address database


If you need to access the orders we have suck due to address mistake:


  1. Sales tab
  2. Incorrect address
  3. Store - Select related market / country
  4. Type - filter the type of orders that should be presented







Correcting the address


Access the related order page.


  1. Click to Edit the Shipping address
  2. Review & copy the original information provided by the customer (check Investigation steps below)
    Change the existing address according to the format in google while also changing the special characters (if any)
  3. Save Order address
  4. Set delivery note under the order details box
  5. Paste the original address provided by the customer and Update note


Please note: After updating the address you need to refresh the barcode (to release from the stuck). Check the How to: Packing Slip section in this Magento: back-office article.







Address investigation Steps


  • Copy the original information - as you will have to log a Delivery note in the order details with the original information provided by the customer.
  • Make sure that all special characters are replaced with regular characters
    (ä = ae, ü = ue, ö = oe, ß = s/ss)
  • Copy-paste the existing street address, city, and zip code in google (verify if it is correct)
  • Please note: blank spaces at the end of any information can be the cause of the error. Ensure you remove it.
  • Correct the information as per previous step 2.


Note that the customer’s name and street address cannot exceed 35 characters including the spaces.




Dropshipping API orders


Packing Slips (Address/ Label Errors)


Whenever you need to refresh the status of a barcode (Label error), you can do it accessing the related order/barcode on the Packing Slips page:


  1. Sales
  2. Dropshipping API orders
  3. Packing Slips



To refresh the status and release the order stuck:


  1. Select related barcode/order you have updated the address
  2. Actions: Change status(error only)

    Status: New

  3. Submit


Packing Slips (Attaching a Label)


  1. Search for the related order.
  2. Upload label, add track number
  3. Save







Return Page



Pending Label list



To access the list of Returns and identify the ones you need to create a label:


1. Catalog

2. Product Returns

3. All


Filter:

  • Market-related
  • Shipping Method (label related options)
  • Status: Expected





The label should be created via the related courier portal.



Add label ID



After creating the label based on the Return request information, you should add the label (tracking id) to the return related (comment) and include pick-up date.




Return page


  • Comment section: Add tracking ID number (label) & pick up date
  • Save






Update Return status

Expected to Completed


If there is no API available (integration that automates the change of the return status from Expected to Initiated), sometimes we may need to change the status manually. For that, under the return page:


1. Status: update it to Complete

2. Save





Re-trigger return (API)


Certain scenarios will require you to restart a return. As an example, 2 Men handling / DHL return is triggered by the API calls, and Magento automatically transmits the request to our partner.


To ensure you initiate the return (again) whenever needed you should change it "Customer pick up status (collection).




Steps: Under the return related:

  1. Change the customer pickup status from "DHL Delivery commissioned" to "Pending"
  2. Save.







Warehouse Inventory






Once opening the Warehouse Stock page, you can narrow down your search by adding the SKU number to its search. Also, you can add a filter related to the Store (market).


From it, you can check the combination between warehouse & suppliers and identify the number of SKUs available.




You may find scenarios, where one warehouse is out of the stock. Whenever it happens and you have no information about a new SKU available, escalate to your country's Logistic team for further information.
