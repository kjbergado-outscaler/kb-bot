What you must know before opening a JIRA ticket to Customer Issue Management Team?


1) Before opening a ticket with us, please make sure that your request belongs to one of the Types of requests below:


  • Need to report a Webshop Issue

  • Need assistance with an escalation case


  • Note: for Learning & Knowledge Team, please follow this KB article.

2) Only Supervisors, Team Leaders, Quality Analysts, andTrainers can open JIRA tickets to the CIM Team. If you are an agent, please reach out to your supervisor to address your issue or suggestion.


3) Please make sure that you have checked and followed the process in the KB article before the JIRA is opened.



How can I open a "Webshop Issue" JIRA?


You must use this JIRA everytime you want to report a problem on the Emma website. Examples of issues:

  • Reporting a typo/outdated information with FAQ/product description;

  • Customer reporting an issue at checkout - wrong price, promotional code not applying, wrong information on the FAQs



Make sure the JIRA is correctly filled:


  1. Summary: Date of report (automation? + Short description on the issue type

  2. Description: Make sure you have briefly explained what is the issue you are reporting and why is this considered an issue ADD EXAMPLE

  3. Market: Always add the market you are reporting so that we can understand where should we check the issue

  4. Link: Same as for the market, always add the URL link

  5. Attachment: This is important to help highlighting where the issue is located


How can I open a "Assistance with an escalation ticket" JIRA?


You must use this JIRA everytime you have a ticket where:

  • Customer is threatening with legal actions or to take the case public

  • Claim: +2 supplements already created and no solution

  • Overly complex tickets where process was followed and there is no resolution/stuck

  • No process



Make sure the JIRA is correctly filled:


  1. Summary: Date of report + Short description on the issue type

  2. Ticket Category: Please select the same as in the Sprinklr ticket category - you have a drop down manu with all options available

  3. Description: Make sure you have briefly explained what is the issue you are reporting and why is this considered an issue. Example:

Hey team,

I need your help with the issue above.

(Add chronologic ticket summary. Include a short summary about the process followed and why was the ticket not resolved yet/ what is customer expecting)

Thanks!
