Scope

A fulfilment is effectively how the order is processed and reflects the true status of an order. Creating a fulfilment assigns stock to an order and assigns the order to a warehouse. It is that same fulfilment that is then transmitted to the said warehouse.

After an order is created, it follows a set of “pallet allocation rules” which will determine if the order is created as a pallet:

  • For parcels, a single fulfilment is created automatically, and the order is sent automatically to the warehouse (WH).

  • For pallets, the fulfilment needs to be created manually. The system will then do stock checks and shortly after the pallet will need to be transmitted manually to the WH by the OM. Fulfilment creation and transmission to the warehouse is one of the main chokepoints in pallet OTIF metrics.

    ​

This SOP focuses on the fulfilment creation for pallets. After creation, the fulfilment transmission to the Warehouse will remain Order Management's responsibility.

How to Create a Fulfillment

If there are no fulfilments yet, hit the “Create pallet fulfilment” button.

​

Look at the items and their quantity:

 Sometimes an order will be assigned as a pallet, even if it’s just a very small SKU, like a pillow. These orders can be sent as a parcel.

The rule is: 3 mattresses or below: parcel. 5 pillows or below: parcel.

If you think the order is too small to be a parcel: hit the option “ship as a parcel”.

​

Fill the fields as per the image below. All fields filled in the image are mandatory, not only those marked with an asterisk:

  • “Location” will always be “REN1_FR”

  • “Carrier” will always be “Schenker_heavy”

  • “Shipment reference”: should be the date in format “DD/MM/YY-identifier”

    • You can use as identifier whatever you want, the point is for you to recognize the order.

  • “AVIS number” is the same as “Shipment reference”

  • Dispatch date should be the next business day: if we’re on 02/05/2025 (a Friday), dispatch date is 05/05/2025 (Monday) as per the image below.

It is possible for an order to have multiple fulfilments. This is done by selecting only a limited amount of quantities/SKUs and submitting the fulfilment.

Register the order in the pallet or truck file file

After creating the fulfilment, the fulfilment needs to be recorded.

Recording is important because pallet transmission is a Work in Progress, technically.

The OM controls transmission on a weekly basis.

Where the transmission is recorded depends on the size of the order.

  • Truck/Gigantic orders. Orders with more than 30k€ in value, more than 60 mattresses are generally FTL (full truck load) orders.

    • These orders must be recorded in the Truck file: Offline_FR_Truck_Shipment_File_Version_1.xlsx

    • The following fields are mandatory:

      • Order. If you input any duplicates, the file will flag the duplicate values in Red. Please confirm there are no duplicates.

      • Fulfillment. If you input any duplicates, the file will flag the duplicate values in Red. Please confirm there are no duplicates.

      • Destination address

      • Name (for delivery contact)

      • Contact (for delivery arrangement by telephone)

      • If there are any special topics regarding the order, log them in the column “Remarks / Special Requirement”

  • “Normal” Pallets are every other order, including orders “sent as parcel”.

    • These orders must be should recorded on the Pallet file

    • The following fields are mandatory:

      • Order ref: example: W-0007-FQKBD3. If you input any duplicates, the file will flag the duplicate values in Red. Please confirm there are no duplicates.

      • Fulfilment date: format DD/MM/YYYY

      • Fulfilment ref : example : 16101610-EAAIV. If you input any duplicates, the file will flag the duplicate values in Red. Please confirm there are no duplicates.

      • Sent as a parcel?: yes or empty cell

The remaining columns will be for Order Management and Logistics usage.

Final notes

  • If you think something is wrong or seems suspicious, check with the OM. The pallet process in general is still undergoing development.

  • Things that are suspicious: monstrously big orders (over 100k€) probably lack the appropriate discount code, fulfilment not appearing after you’ve created it and refreshed the page, Fluent error messages.

  • Order from customers of stock depletion should not be fulfilled. These customers are, for the time being, “EO Trading” and “LemonSleep.
