Updating the Auchan order file

Scope

Auchan retailer have dropshipments and warehouse orders that need to be created in the system in order for it to be processed and delivered systematically. This SOP will show the two different processes of manual order creation for Auchan.

​

Key Requirements

  1. Customer Service (CS)

    • Monitor incoming emails from Auchan on a regular basis.

    • Request Backoffice to create the corresponding order in Fluent Commerce.

  2. Backoffice

    • For transmitted pallet orders, process the order accordingly.

    • Inform the Warehouse (WH) of the delivery appointment.

    • Add the order to the Auchan order tracking file.

    • This file will later be used to send the PODs (Proof of Delivery) in a separate task.

Instructions

Task for CS

  • Auchan orders are received for manual creation via two different email addresses.

  • Customer Service (CS) must create a ticket with Backoffice using the following link:
    Single Order Creation

  • The ticket must include:

    • All order details (e.g., SKU, quantity, delivery date, etc.)

    • The documentation sent by the retailer (Auchan) as an attachment

​​

​

Task for BackOffice

Order Creation

  • Review the order sent by the retailer (must be attached to the Jira ticket).

Minimum qtys of 20 or more pillows or qtys of 6 or more mattresses. (for pallets)

  • Open the PDF file containing the Purchase Order. From here you can take some information:

    • BU-key - BU-TTRTM

    • External order number - starts always with a 9*

    • SKU + Qty

    • Appoitment delivery - Data de Entrega

Updating the Auchan Order File

  • Once the order is created, add it to the file:
    Status_Orders_Auchan_New.xlsm

  • The following fields must be filled in:

    • Order Date

    • Internal Order Number

    • External Order Number
