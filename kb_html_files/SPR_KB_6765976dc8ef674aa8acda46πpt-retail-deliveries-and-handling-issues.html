Order Status (WIMO)

The estimated dates from Magento and FluentCommerce should not be given. They may be incorrect, and we cannot guarantee delivery on that day.

If the retailer contacts us to know the order status, we need to check if it is with the carrier or in the warehouse to know whom to contact.

If it is with the carrier, we can check on their platform if they have a tracking link and if it is updated. We can send the public link to the retailer.

This file contains the delivery times for each item from the order creation: Retail Lead Times.xlsx. Outside these times, we can consider that there was a delay. 

​

FOR REFERENCE (internal information): 90% of orders delivered by GLS are delivered in less than 4 days from the “grabado” status, so when checking the carrier’s platform and seeing that 2 days have passed, you should follow up with the carrier.

Orders from the islands can take up to 2 weeks to be delivered. We can give this information to the retailer: “Uma vez que se trata de uma expedição marítima, o tempo de entrega é ligeiramente mais elevado.”

If we receive an email with a list of several orders, just indicate the current status of the order. If you notice any issue in the tracking, provide the necessary action and do not just inform the order status.

​

Supplements

Whenever it is necessary to make a supplement (especially for older supplements) or create a new order, ALWAYS confirm the final customer’s details with the retailer (address and contact). If we do not get a response within 2 hours, the agent should call to ensure we have the correct details for the shipment to be made correctly and should follow up until it is delivered to the customer.

Whenever Maigerlis Correia needs to make a supplement, she will leave a note on the order with the reason, so we have easy access to the information and inform the retailer accordingly.

Whenever you need to make a supplement for a canapé or part of a canapé, always open a Jira for the WH, asking to change the carrier to Mobel or it will be assigned to XPO (it is automatic). This is a mandatory step.

​

Assembly Service

Assembly always occurs for canapés or upholstered bases. The service is included, and the carrier must do it.

Order in “grabado” status

Sometimes when the order is in this status on the carrier’s platform, it means it is still in the warehouse.

If we want more information, we need to write to the warehouse. If it also does not have a barcode or carrier assigned in Magento / Fluent, then we contact the WH (warehouse).

​

Order with delay or issue

  • If the order is within the lead time and the customer asks for the status, we do not need to follow up until delivery.

  • We only do this when we verify that it is outside the deadline or has an issue because we need to investigate with the carrier/warehouse. We should keep the ticket on hold until it is delivered, then inform the retailer and send the POD.

​

Discrepancy between carrier and retailer

Retailer says it was not delivered / carrier says it was. We should contact the warehouse.

​

Customer refuses delivery

The carrier may inform that the customer refused the delivery.

We should email the retailer to understand the reason for the customer’s refusal. Two situations can occur:

  • Customer refused and does not want to receive, so the store must be informed.

  • Customer wants to receive, and we have incorrect information.

Customer absent

We may have cases where, according to the carrier’s platform, the customer was absent when they attempted delivery, and the retailer wants a new shipment.

We should email the carrier to make a new shipment.

It is advisable to inform the retailer of what we will do, and if we do not get immediate responses, inform that we will get back in touch. After the carrier responds, we inform the retailer again.

If the customer was absent twice, we inform the retailer and ask if the customer still wants the order.

We can ask the carrier to make a third delivery; they will say there is an associated cost, so we notify Maigerlis in a note on the same ticket, and she must authorize it for us to confirm with the carrier.

​

Order returns to warehouse / lost / damaged before delivery

Due to delivery failure by the carrier or loss, the delivery may return to the warehouse and never be delivered. If the customer still wants the order, we make a supplement in Magento.

If the order is in Fluent, we create a JIRA to BO to create the order because the first one was not invoiced.

Whenever we create a JIRA to make a new order because the original was not invoiced in Fluent, we must return the original order. This way, we know it will not be invoiced in the future.

Lost order

If we write to the warehouse and the carrier and no one has the order, we can consider it lost.

We make a logistic claim and supplement. If it is in Fluent, and the order has not been invoiced, we must ask BO to create the order.

Order in ParcelShop (GLS)

If the order is delivered to a GLS Parcel Shop and the customer requests a new home delivery, we must indicate that it is not possible, as the order has already been delivered to the Pick-up point and cannot be changed.

The order remains in the Parcel Shop for 10 days and is automatically returned after this period.

In this case, we can offer a new shipment, but we can only do so once the order is returned.
