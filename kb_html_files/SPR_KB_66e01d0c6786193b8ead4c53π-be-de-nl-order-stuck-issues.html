Please check here the Globalobal article on how to place and check a return


Troubleshooting a Stuck Return


A return being stuck can be caused by a lot of various factors since a return goes through various systems and people for it to be completed. To properly troubleshoot a stuck return, the following can be done:


1. Checking Customer and CS Agent conversation in Freshdesk


Link to Freshdeskesk: here

In order to understand the situation of the return, the conversation between agent and customer needs to be checked. In order to do this, search the order number of the order and read through the conversations associated with the order.

Here are some cases to watch out for:

  1. Customer requested a return.

    1. and is still waiting for return to happen.

    2. and probably changed their mind and did not complete the return.

    3. and is still waiting for a delivery before performing a return.

    4. and the return is complete but still waiting for the refund.

  2. Customer did not request a return.

    1. but waiting for a refund.


This information will help understand what should happen in our systems.


2. Checking a return in Fluent

  • Go to Fluent and search the Order Number. Open the “Related” tab and check the Returns

  • From there, you will see all the returns created for the order. Click on the “Return Order Ref” to view the return.
  • If there is no return created, please create the return..

Understanding Stucks (Known Stuck)


Stuck In Fluent

  1. Return Not Created

    1. Return cannot be created

      1. Fulfilment Status is not complete

        • Sometimes, the customer already received the item, but in FluentCommercem the item is not yet delivered.

        • If the fulfilment status of the item is one of the following statuses, then the return cannot be created.

          • Waiting_for_stock,

          • SUBMITTED_TO_FL,

          • RECEIVED_BY_FL,

          • Waiting for Stock at FL,

          • PICKED_PACKED_BY_FL,

          • READY_TO_DISPATCH

        • For these cases,

          • File a ticket with Fulfilment Stucks Concerns: Fulfillment Stuck Concerns - Order Fulfillment Concerns - Jira Service Management

            • When they fix the fulfilment status, the return can be created normally.

          • if the refund is escalated file for a manual refund in the OTC Portal: Order-to-Cash Portal - Jira Service Management

      2. Fulfilment Status is complete, but return cannot be created.ted 

        • no return created, but returnable_qty = 0,

        • For these cases, go to this Return Escalations link and file a ticket

  2. Return stuck at Return_Created status.

    1. A Return was created wrong

      • Return without refund instead of return with refund created.

        • In order to fix this, the solution is to cancel and recreate the return. The following should be done: Canceling and Recreating Return 

      • Wrong items selected

        • In order to fix this, the solution is to cancel and recreate the return. The following should be done: Canceling and Recreating Return 

      • Too many items selected

        • Same as above, In order to fix this, the solution is to cancel and recreate the return. The following should be done: Canceling and Recreating Return

      • Return was not needed

        • Happens if customer never wanted a return in the first place, but just a refund (maybe they never received the item)

        • If a return was created even though it was not needed, please follow: Canceling a Return

    2. Return stuck in Return_Created

      • Label/Drop-off

        • If there is no tracking number, check Freshdesk and search the order number, try to check the conversation if there is a tracking number, or if the customer already said they completed the drop-off.

          • if there is a tracking number, put the tracking number into Fluent Commerce and move return to organised: Marking a return as complete

          • If customer already completed the drop-off, mark the return as complete: Marking a return as complete

        • If the customer is actually asking for a label,

          • For DE go through the Return Widget on the website. Customer will automatically receive their label through email

        • If there is no escalation, just cancel the return: Canceling a Return

      • 2MH

        • If a return is 2MH, please skip this as of now.

  3. Return stuck at RETURN_ORGANISED status

    1. already completed

      • If a return is in RETURN_ORGANISED but customer says the return is already complete, we need to complete the return. In order to do that, follow: Marking a return as complete.

    2. Without Tracking Number

      • DHL Label

        • If a label return is in organized but does not have a tracking number, check freshdesk for the tracking number. If there is still no tracking number:

          • if greater than 60 days old, cancel the return.

          • if less than 60 days old, do not do anything

      • 2MH

        • If a return is in Organized but does not have a tracking number, please skip this as of now

    3. With Tracking Number

      • DHL Label

        • If a return is in RETURN_ORGANISED and has a tracking number,

          • if greater than 60 days old, cancel the return.

          • if less than 60 days old, do not do anything

      • 2MH

        • If a return in in RETURN_ORGANISED and has a tracking number, please leave is as of now

4. Refund Stucks

  • Refunds topics should be raised to the Refunds team. You can file tickets through their portal: OTC Portal

Fixes for stucks


Canceling a Return

In order to cancel a return:

  1. Open the return

  2. Click the cancel button on the upper right



Canceling and Recreating a Return


In order to cancel and recreate a return:

  1. Open the return

  2. Take note of all the details needed for when to recreate, including:

    1. SKU and quantities

    2. Return reason and condition

Cancel the return by clicking the cancel button on the Upper Right


Recreating the Return:

  1. Go to the Order

  2. On the upper right, click the three buttons and depending if with refund or not, select “CREATE RETURN WITH REFUND” or “CREATE RETURN WITHOUT REFUND”

  3. Input the details noted from the previous return

NOTE: DO NOT MOVE THE RETURN TO ORGANISED. We have a robot that will do this automatically, and put the tracking number of the return.


Marking Return as Complete


If return is at status Return_Created:

  1. Go to the return. On the upper right, there is a button “MARK RETURN ORGANISED”. Click this button and confirm.



Once/If a return is at organised:

  • Click the box under “Mark complete” for the items.

NOTE: All items in the return need to be marked as complete for the return to be moved to complete. This is also required in order to issue a refund.
