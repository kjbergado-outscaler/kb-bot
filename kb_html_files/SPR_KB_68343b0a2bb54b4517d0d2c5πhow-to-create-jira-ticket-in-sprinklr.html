What is the JIRA integration on Sprinklr?

The JIRA integration is a new feature that connects Sprinklr with the JIRA system, allowing Agents to open JIRA tickets directly from Sprinklr, eliminating the need to open the JIRA Portal separately.

Check below the principal changes & key rules of using the JIRA integration:

  • Whenever a JIRA ticket is created, it gets linked to the case in Sprinklr;

  • Any agent will be able to see and work on the same JIRA ticket;

  • Any action needed on the JIRA ticket can be done directly via Sprinklr (check comments, attachments, and status);

  • Whenever an Emma team works on the JIRA ticket and send it back to Agents, the tickets goes automatically to the queue and will be assigned to the next available agent.

Handling the case with a JIRA ticket created

-> How to add comments and attachments:

After a Jira ticket is created, Agents can collaborate directly with Emma internal team through Sprinklr using the widget "Case Collaboration". This means you don’t have to open Jira separately to reach out to the Emma Team resposible for the ticket.

Any comment or attachment posted by you in the Sprinklr case will sync to JIRA ticket, and vice versa.

You can comment, attach files, or request more information for the Emma team handling the Jira ticket via "Case Collaboration", on "Add a Case Note" section. In the Sprinklr case, go to Case Collaboration, click Add Note, write your comment and post it. To add attachments, click on the clip icon to the left, select your file and send the message.

-> How to open attached files:

For the situations when the Emma team needs to send you an attachment, you will also be able to see it via the “Case collaboration” section. However, you must follow the below steps to open correctly the file/image:  

  1. Go to the Sprinklr case; 

  2. Find the message that contains the file link in blue color (it usually starts with https://...).

    ⚠️ If the agent clicks the link directly, it may show a "blocked" message — hence copying it manually is required. 

    ​

  3. Click Edit to view the raw link: 

    ​

    ​

    ​

  4. Now look for the part of the link that starts with: 
    https:// 
    and stops right before the part that says |smart-link]; 

    Example: 
    If you see: 
    [https://emma-files.com/doc123456|smart-link] 
    ➡️ Copy only: https://emma-files.com/doc123456

FAQs

Q. How can I check previous Jira tickets?

  • Only one Jira ticket can be linked per case at a time.

  • You can create another Jira ticket after the case is resolved.

  • You can also unlink a ticket and link a new one to the same case if needed.

  • Once a Jira ticket is unlinked from the case, you cannot see the summary of it, but you'll still see the notes in the case collaboration section.

    -> If you need to check something from previous tickets:

  • Go to the Case Collaboration section: all past tickets reference numbers and comments remain visible there.

  • Agents must share the ticket number with Supervisor

  • Supervisor can access the ticket content with the reference number.

Q. How will I know if the Emma team replies to my ticket?

You do not need to check Jira manually. When the internal team replies, their message will appear in the case collaboration section in Sprinklr. The case will also return to your queue automatically.

Q. How does Jira Ticket status work?

​

For Agents, after the JIRA ticket is created correctly and the ACW is applied with Case Status as "On Hold" (as per the process), no extra action is needed. When the internal team works and replies to the JIRA ticker, the Sprinklr case gets updated and goes automatically to the queue.

Close the case only once the issue with the customer is fully resolved.

Q. What should I do if the Jira ticket is not submitting?

Check that all mandatory fields are filled out correctly. This includes the Application, Reporter, Summary, and any other required fields for your request type.

Q. What should I do if the attachment won’t upload?

Make sure you are using a supported file type such as PNG, JPG, PDF, XLSX, or DOCX. If the file size is too large, compress it and try again.

Q. What if the Jira ticket is not visible in Sprinklr after I create it?

Click on "View Case Details" again to refresh the view. Also double-check that the Reporter field was set to the correct email: sprinklr-agent@emma-sleep.com.
