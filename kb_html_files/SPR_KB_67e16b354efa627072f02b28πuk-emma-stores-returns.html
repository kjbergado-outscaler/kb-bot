This process is done when the customer wants to return a product within their 200-night trial period.

Confirm Return Eligibility

  • When a customer requests a return, first verify their eligibility;

  • Check if the return request falls within the store’s policy (e.g., return window, product condition).

  • If eligible, start the Returns process. If not, kindly advise the customer that it's not possible to proceed with the return and refund.

Please advise the customer that if the product is faulty, we can always proceed with the warranty analysis.

  • Fill out all requested information

​

Once the collection is confirmed, the column “Requested (FOR COLLECTION TEAM ONLY)” will be marked with a “c” and a green background, indicating that the item is ready for pickup.

​

​

Inform the customer that collection must occur before the refund process begins. Customers often assume refunds start upon pickup. This clarification helps manage expectations.

Track the Collection

  • Check the tracking system (Rhenus portal) to confirm when the item has been scanned at the depot.

  • Customers may also receive a confirmation email with tracking details.

  • If needed, request customers to inform you when their item has been collected.

Refunds cannot be processed until items reach the depot.

Process the Return and Refund in Magento

  • Click in Return Products;

​

  • Fill-out the information accordingly:

    • Return Products: select the returned product(s)

    • Quantity: add the quantity of returned SKUs

    • Refund to the customer: mark if the return should be followed by a refund. If it’s just a collection case, select “No”.

    • Warehouse Address: make sure to select the correct warehouse if the product is opened.

    • Reason: select if it’s a Delay, Product Claim, Logistic Claim, etc.

​

  • Click Save in the top-right corner.

The return will then be created and will appear in the Order page, as Product Returns.
