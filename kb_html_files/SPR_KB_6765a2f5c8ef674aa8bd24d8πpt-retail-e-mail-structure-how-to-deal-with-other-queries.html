Email Structure

  1. Correct use of phrases like: "Votos de um ótimo dia / Desejamos um ótimo fim de semana/ etc" ----> These should only be used from now on when the initial issue for which we were contacted is resolved, so you can send these phrases in the last email of the entire interaction.

  2. Do not apologize for returns (100 nights).

  3. We should not apologize more than once in the same email.

  4. We can tell the retailer to wait 48 hours for our response when we forward the case to the carrier or warehouse.

  5. When we have a complicated situation with an order that has been unresolved for some time, in the email we send, we do not need to include the phrase "Obrigado por nos contactar", but can go directly to the point.

  6. Transparency: Always provide as much information as possible. For example:

    • If a retailer contacts us because they want more information about an order placed three weeks ago, we should give a complete response. If we do not have a delivery forecast or tracking link, we can say: “verificamos que a encomenda suplementar já se encontra em preparação no nosso armazém, e já temos o número de tracking ZZZ, no entanto o link para o seguimento ainda não se encontra disponivel, assim que tivermos a informação disponível voltaremos a contactar".

Ticket Resolution

When resolving a ticket, check if the issue for which the retailer contacted us is indeed resolved.

The maximum resolution time is 24 business hours, not calendar hours. If there is a situation that prevents us from closing the ticket within that period, it will not impact the evaluation grid. However, we must check and update the ticket each day. If there is a failure in this regard, it may impact you.

 

​

Escalations

To escalate a situation, first check the case with the supervisor If we cannot resolve it, escalate to Maigerlis if it is an order management issue, in a note on the FreshDesk ticket, and include supervisors email.

 

​

End Customers

If we receive a call from end customers - we should inform them to contact the store where they made the purchase directly.

If we receive an email from an end customer - transfer it to the PT team.

End customers are not supposed to have our contact or email.

​

Tickets transferred by the D2C Team

Whenever we receive a ticket from D2C that should be handled by them, transfer it back to the PT group.

 

​

Duplicate Emails

If we receive two emails about the same situation, we have two options:

  • If they are exactly the same emails, sent by the same person, then we can merge and resolve, or leave a note “Situation being handled in ticket --”.

If they are different emails, sent by different people, but about the same situation, we should handle the situation in one of them and respond in the other ticket that we have already received the request and it is being handled.
