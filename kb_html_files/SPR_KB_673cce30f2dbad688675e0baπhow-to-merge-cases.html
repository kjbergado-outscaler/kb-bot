​When is it possible to Merge cases?

​Why should cases be merged?

How to merge cases (step-by-step)

When is it possible to Merge cases?

1) On Email cases ONLY.

1.1) It is NOT possible to merge Phone and Live-chat cases. For those, you have to handle and close each one of them. If during the live interaction, you find that on the "historical cases" that the customer request was already addressed and solved, please inform the customer about it and then finalize the interaction.

2) On the emails cases sent from the same customer with the exact same request and about the same order number.

How to merge Email cases (step-by-step)

Step 1: Identify if Merge is applicable

​

After checking the Emails cases, you find that indeed the same customer has 2 emails cases (or more) open with the same request related to the same order, you can proceed with the merge.

​

Step 2: Select the option "Merge Cases"

The merge logic is: the new/open case is the one to be merged with the old/first email case. So, you need to open the old case, click on the 3 dots and then on the option "Merge Cases".

On the example below, the case #2422376 (circle in green) is the new case; and the case #2366362 (yellow circle) is the old/first email case.

Step 3: Search and selected the case to be merged

​

After clicking on the button, a new page will appear and there you can see the old/first case on the right side. And on the left side you have a search bar in which you have to type the case number and click on enter.

​

Following that, the case will appear for you on the list and you must select it on the box and click on the "merge" button.
