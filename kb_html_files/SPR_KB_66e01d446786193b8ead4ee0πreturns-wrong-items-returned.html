Wrong items Returned to Emma

You may face a situation where the customer requests to return an Emma product but instead, they send another item back by mistake. If this happens, the customer may contact our CS line requesting support retrieving the item returned wrongly.

Please check below how to proceed:

First, you should check in FluentCommerce the return status and if the package:

Situation 1: Has been picked up by the carrier but not yet at the warehouse \ returns operated by charities;

Situation 2: Already at the warehouse.


What to do if the package is still with the carrier (or charity returns)


For situation 1, when it is still on route to the Warehouse, the customer needs to contact the carrier\charity directly. The carrier\charity will then check if it can still be retrieved. Only customers are entitled to recover it from the carrier, as they are the ones listed as sender of the parcel and only the sender can do request it.


What to do if the package is already at Emma's Warehouse


For situation 2, if the package is already returned and stored in the warehouse (and there is nothing the carrier can do) the customer must contact us. The agent must then open a Jira ticket in the CE Portal so that the market's Performance Manager can reach to the W2D team and try to trace the parcel at the warehouse.

​

The Ticket must contain the following information:

- Name on the package;

- Tracking number used to return the package;

- What item was returned instead of the EMMA products;

- Invoice of the product returned;

​

Ideally, the customer must provide some proof that the item was wrongly returned to us (for example: if swapped label, a confirmation from the other company that they received an Emma product for example). If not possible, please leave that information on the ticket.

Jira Portal link ("Order Issue"): Order Issue - Customer Excellence Team - Jira Service Management (atlassian.net)
