In this article, you can find information about which Macros are available for CS Agents to use and what is the functionality of each one of them.

A Macro in Sprinklr is an automated workflow that allows users to perform repetitive tasks quickly and efficiently. Check this article to understand the logic of a Macro on Sprinklr: HERE

​

Below are the Macros available for CS agents currently:

  1. Assign to Me: Only to be used with the supervisor's approval. It is used when the TL is reassigning the cases to them from the agent's bucket if the capacity of the agent is overflowing. It is also used when the agent is creating an outbound email and the case needs to be assigned to them or if the case is unassigned to them automatically by the system. 

    ​

  2. ​Change LOB/Market/Language or unassign: (use this with supervisor's approval). It is used when the CS Team lead or agent needs to reassign the case to the correct market or country.

    ​

  3. Create JIRA: NOT IN USE FOR THE MOMENT. And, please, don't use it.

    ​

  4. No Response Required: To be used in Spam cases. This is used in case the customer's inquiry is already answered by a different agent on a related resolved Case.

  5. Trigger ACW: To apply case tagging and case status. To be used in exceptional cases the button “Trigger ACW” does not appear on the top right.
