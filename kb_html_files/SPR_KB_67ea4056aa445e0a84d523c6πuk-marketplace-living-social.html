Daily Order Status Check and Tracker Updates

Important Note:
The tracking task and order creation tasks, outlined below, are done by the Backoffice team. They are not required to do from the CS! Also, soon The SOP of these tasks will be archived since soon there will be Channel Engine integration which performs these tasks automatically.

One thing that still is now required from CS, and perhaps will not be needed in the future, is sending every Tuesday and Friday and email to Living Social with the update on the latest orders which got delivered.

  • Open the LivingSocial Tracker and check for orders created the previous day.

  • Identify orders marked as delivered (green markings) and update their status.

  • Search for each order in the Master Tracker and tag them as delivered.

Order Creation Process

  • Log in to the LivingSocial Merchant Portal (similar to Wowcher but with different credentials):
    Portal URL: https://myaccount.wowcher.co.uk/myaccount/merchant
    Username: livingsocial@emma-sleep.com
    Password: Emma2024

  • Navigate to the Live Deals section.

​

​

  • Download the Redemption File for orders from the previous day.

  • Open the file in Excel and filter for the relevant dates.

  • Compile all valid orders in a separate Excel sheet.

​

​

  • Paste the compiled orders into the LivingSocial Master Template (provided by Murat).

​

​

​

Ensure all fields are correctly populated. If there are missing values or errors:

  • Validate Deal ID and correct any formatting issues.

  • If the Deal ID is incorrect, coordinate with LivingSocial or Murat for resolution.

  • Orders with unresolved errors cannot be created.

​

​

  • Save the completed order file in the LivingSocial Orders – Inbound folder, using the naming format: LivingSocial_Orders_YYYYMMDD.csv.

​

Uploading Orders via FileZilla

Log in to FileZilla

  • Host: sftp.ecom.emma-sleep.com

    • Port: 22

    • Username and Password: Provided via email (Emma SFTP Access)

  • Navigate to the Inbound Folder.

  • Drag and drop the LivingSocial order CSV file into the folder.

​

​

  • Wait for the successful upload confirmation.

  • Navigate to the Feedback Folder, download the processed file.

  • Verify errors:

  • Missing SKU → Reupload only the erroneous order.

  • External Order already created → Do NOT reupload; check Fluent Commerce.

  • File format issues → Ensure CSV UTF-8 encoding.

  • Reupload attempts → Always use a different file name.

​

​

  • Confirm the uploaded orders in Fluent Commerce:

  • Search by Order Number or External Reference ID.

  • Ensure all orders appear in the system.

Tracking Order Status

A. Non-Refurbished Orders – GLS Tracking

  1. Log in to the GLS Portal

    • Portal URL: GLS Tracking Portal

    • No login required.

  2. Search using the tracking number (usually starts with 2, 1, or 5).

  3. If the order is marked as delivered, update the LivingSocial Tracker.

  4. If tracking information is missing, escalate to GLS support via email.

  5. For Non-Refurbished orders, you may use the tracking number provided by GLS to check the status of the order. This can be found in fluent commerce under each ticket's fulfilment section.

​

​

​

  1. Log in to Stream:
    Portal URL: https://www.go2stream.net/stream/live/core/view/Login.php
    Username: EMAEMMASLE
    Password: Emmapassword1@

  3. Cross-check the post code with tracker information.

  4. If the order is delivered, update the LivingSocial Tracker.

Customer Inquiries and Issue Resolution

  1. Access the Customer Support Email System

    • LivingSocial forwards customer inquiries regarding orders.

    • Open Sprinklr to handle customer tickets.

  2. Locate the customer’s order using the voucher code.

    • If found, verify its status via Fluent Commerce.

    • If missing, check Magento or the LivingSocial Redemption Files.

  3. Common Issues & Resolutions:

    • Unlocated Orders: Check the tracker. If missing, escalate to LivingSocial.

    • Order Not Created: If not found in Fluent Commerce, verify if the redemption details were received.

    • Faulty Product Received: Handle via the standard return policy.

  4. Verification Process:

    • Customers must provide a redemption code and screenshot before assistance.

    • If missing, request verification before proceeding.

    • Use the CHEAT SHEET for standard response templates.

  5. Return & Refund Policy:

    • Non-Refurbished Orders:

      • Follow LivingSocial's 14-day return policy.

      • Refunds processed after the return is confirmed.

    • Refurbished Orders:

      • 6-month warranty applies.

      • Thermo warehouse handles processing.

    • Refunds take 7-14 business days.

  6. Finalizing Tickets:

    • Once an issue is resolved, update Sprinklr with the final response.

    • Apply the correct status tag: Resolved, Waiting for Response, or Escalated.

Cancellations

Cancel in Fluent Commerce (FC):

  • Orders must be cancelled before they are dispatched to Fulfillment Center (FC).

  • Go to the pending order → Click "Cancel order item" → Input quantity and reason → Submit.

  • Add the cancellation information in this tracker (Cancel Sheet): MKP CS TO OM request for channel engine support.xlsx

If cancellation in FC is not possible, CS must fill out the Cancellation Request File and notify Shimul.

Cancellation Request File

  • Access the file here;

  • Use this file when cancellation cannot be completed via FC and notify Shimul;

  • He will then proceed with the cancellation.

  • Fields to fill:

    • Order number

    • External order number

    • SKU to cancel

    • Fulfillment reference

    • Return to sender status

    • Tracking number

    • Carrier code

    • Reason for cancellation

  • Color Codes:

    • White: Not processed

    • Orange: Ticket raised

    • Green: Completed

Note: If an order has multiple items that need to be cancelled, fill out one line by Fulfillment/SKU.
