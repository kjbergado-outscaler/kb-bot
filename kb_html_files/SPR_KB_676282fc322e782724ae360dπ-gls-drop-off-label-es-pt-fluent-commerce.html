Note:

  • For accessories Fluent Commerce orders, these returns must go via the Return Portal, see instructions at the top of this page.

  • Packed mattresses, shipped by GLS, for Fluent Commerce orders must be manually created to GLS.

Customer must use the Returns Portal. The return is created and processed automatically. Only in case there is an issue with the return creation via the portal, you can still create returns for accessories manually.

How to verify if the return was created correctly

  • The customer contact details are correct

  • The return status must be set to 'Created'

  • The pick up date must be left empty

  • The carrier field is populated with: GLS_ESPT

Note: The return must never be manually set to 'Organized'. If the return status is set to 'organized' manually, the return will not be recognized by our platforms, and it will never be forwarded to the return carrier. Please ensure the return status is 'Created' after you created the return.

How are status updates handled for the return

  • Status updates for Fluent Commerce are automated.

  • If needed, you can confirm in Parcel Lab for the tracking status.

  • If nothing is showing, GLS Portal should also have the updates.

How is the Refund Triggered?

Note: This process applies to returns within the trial period.

​

To process the refund, the order must be completed: In the OMS (Fluent Commerce), the automation will mark the return as 'complete' to trigger the refund automatically. No input needed from an agent.

Note:

  • No Proof of Payment: Do not request proof of payment or product pictures from customers or carriers.

  • Unresolved Tickets: For tickets open for more than 14 days, create a JIRA ticket in "Order issue" at the Customer Excellence Portal .

Refund Timelines:

The refund time starts when the refunds team approves it (up to a week after return completion).

Payment Provider Timelines:

  • PayPal: 1-4 working days

  • Amazon Pay: 2-7 working days

  • Bank Transfer: 2-9 working days

  • Aplázame: 2-9 working days

  • Credit/Debit Card: Up to 4 weeks

Follow-Up: If the refund exceeds these timelines by more than 5 days, file a 'Refund follow-up' Jira ticket with the AR D2C team via their Jira portal.

When can this return ticket be considered 'Resolved'

Once:

  1. The customer reported the return was performed

    OR

  2. The status of the return has been updated automatically in the OMS to 'COMPLETE'

    OR (in case of refunds)

  3. You have informed the customer of the expected timeline for the processing of the refund.

    ​

The return can be considered 'resolved' and you can close the case.
