Creating returns with ThermoProducts

Note: You will need to create the return both in the OMS, as well as via the provided Microsoft Form

Note 2: This process is the same for both Magento and Fluent Commerce returns

Step 1: Check for Active Returns: Ensure there is no active return in Fluent Commerce for the same items (the customer might have already created a return via the portal).

Step 2: Create Return in OMS (Fluent):

  • Carrier field: Fluent Commerce: TP_IE

  • Pick up date: Leave empty

  • Return status: Do not move to 'Organized'

Step 3: Fill in this Microsoft Form*: https://forms.office.com/e/ymp99BPDPj

This needs to be filled in for every item that is in the return! So if there are 3 items in the return, you have to fill it in 3 times, even when the item type is the same.

  • Order number: <Paste the order number>

  • Customer name: <Paste the customer's full name>

  • Address: <Paste the customer's address>

  • Post code: <Paste the customer's post code>

  • Contact no.: <Paste the customer's phone number>

  • Email address: <Paste the customer's email address>

  • SKU: <Paste the SKU of the item that needs to be returned>

  • Product description: <Paste the name of the item that the customer is returning>

Step 4: Check Return Status: Check the return status on the planned pickup date or after 10 working days on the Go2Stream tracking page to see if the order has been completed.

How to verify if the return was created correctly

  • The customer contact details are correct

  • The return status must be set to 'Created'

  • The pick up date must be left empty

  • The carrier field is populated with: TP_IE

How are status updates handled for the return

  • Status updates for Fluent Commerce and Magento are not automated.

  • You (the agent) must monitor the return status on the email thread with the carrier on the planned pick-up date.

  • Follow up after one week to ensure the return is processed.

  • Update the return status in the Order Management System (OMS).

How the refund is triggered

When to Start the Refund Process: Keep an eye on the status of the return in Thermo's Portal. Once the item(s) has been picked up, you need to manually complete the Order in the OMS (Fluent Commerce). Markingthe return as 'complete' will trigger the refund automatically.

If a customer reaches out to you mentioning that the product was picked up, you need to check the status of the return on Thermo's portal.

Refund timeline: The refund time starts once the order is completed in OMS. Our refunds team (ARD2C) usually releases the refund within 2 working days. After that, it will depend on the customer's own payment provider how long it takes before they see the refund in their bank account.

Note:

  • No Proof of Payment: Do not request proof of payment or product pictures from customers or carriers.

  • Unresolved Tickets: For tickets open for more than 14 days, create a JIRA ticket in "Order issue" at the Customer Excellence Portal .

When can this return ticket be considered 'Resolved'

Once:

  • The customer reported the return was performed

    AND

  • You have updated the status of the return in the OMS to 'COMPLETE'

    AND (in case of refunds)

  • You have informed the customer of the expected timeline for the processing of the refund.

The return can be considered 'resolved' and you can close the case.
