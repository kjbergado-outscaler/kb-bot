Searching for Orders

Internal Order Number Search

  • To search using an internal order number, click “Add a filter”, select the “Order Ref” field and enter the order number.

External Order Number Search

  • If the order number is unavailable, use the "External Order" field.

    • Enter the external order reference and apply the filter.

​

Searching with STRG + Alt + K

  • The most efficient search method is using the "CTRL + Alt + K" option, or simply clicking in the magnifying glass on the top-right corner.

    • This method shows both exact matches and near-matches (for cases where a number is missing or slightly different).

Order Overview

Once an order is located, you’ll be able to see all relevant information:

​

​

Order Lifecycle Bar

Here, you can see the order status.

​

Orders can be canceled if they are in Pending or Booked status. Cancellation depends on whether the order has been transmitted to the warehouse.

​

You can also see:

Summary

  • External Order Number: This is the Retailer’s order reference.

  • Order Status: Displays the current status.

  • Creation Date: Date when the order was created.

  • Shipment Type: Whether the order is partial, truck shipment, or bulk shipment.

Retailer Info

  • Retailer: Represents the Country or Market where the order has been placed.

Customer and Delivery Information

  • Customer Details: Includes the customer's store, phone number, and email address for notifications.

  • Delivery Information: Review the delivery address and verify if it’s the correct destination (e.g., whether it goes to the graphite warehouse or another location).

Fulfilment List

  • Each item has an associated Fulfillment Number, which can be searched to track specific shipments. Here, you have an overview of this information.

Order Items

  • This section provides a snapshot of the items included in the order, allowing us to review product details such as the SKU, quantity, and price.

Order Total

  • For B2B orders, the overview only shows the value of the order (i.e., the total amount), but does not provide detailed visibility into what products are included in the order.

Notifications

  • This section helps to keep track of what has been communicated to the customer regarding their order, including order confirmations and invoices.

Tabs

Fulfilments

Clicking the fulfillment number (ID) will display detailed tracking and shipment information. You can also click on the Tab directly to access this information.

​

​

Articles

  • The Articles section provides the tracking number, allowing users to check shipment progress via external systems (e.g., GLS).

​

​

​

If there’s more than one item in the order, you need to check them individually to get the correspondent tracking number.

Related

On this section, cancellations, returns and supplements are tracked.

  • If no returns or cancellations are listed, it means the order was completed without issues and this section will be empty.

  • If a return is in progress, the related orders and their status will be shown here.

  • Any supplement orders will be listed here.

​

​

​

Returns should not be searched directly. Instead, navigate to the order and use the Related section to access return details, as returns are linked to specific orders.

Comments

All actions taken on an order (e.g., cancellations, status changes) should be documented in the comments section. This ensures that there is a record of the reason for any changes, which can be useful when customers inquire about order issues.

​

​

Transactions

The Transactions tab is not a relevant section for our workflow in Offline, as there is no significant information provided on this section that the team needs or utilizes.

​

Attributes

The Attributes section is mostly used to find invoice numbers. This is crucial for tracking and matching invoices to specific orders. It is also used to track credit memos.

​

​

Activity

The Activities section records all actions performed on an order. However, there may be instances where actions are recorded under "automation" or without specific user names. It’s important to be aware that automation or system-generated actions may appear under generic labels, such as Rubix User.

Supplement Creation

When to create a Supplement Order?

In the scenarios that the customer is entitled to have a Replacement, or when it's a case for an Add-on. 

For reasons Replacement, we consider:

  • Logistic claim for wrong product sent (from Emma's side);

  • Logistic claim for lost/stolen;

  • Product claim (any).

Examples: Customer ordered a 140x200cm mattress but received a 160x200cm; Or if the product if the customer ordered a bed and didn't receive all the parcels for the assembly; Or if the customer gets a product that has a broken foam or a dimension deviation. 

For reasons Add-on, we consider: 

The products are sent to the customer as an added value to the order or original product.

Examples: Sending a pillow as an extra due to a delay case; Or sending a comfort layer on return deflection cases.

When NOT to create a Supplement Order? 

We have certain scenarios in which we SHOULD NEVER create a Supplement Order. Those are the following: 

  • If the customer does not like his SKU or wants to return it during the trial period (exchange the color of the product, for example);

  • If the customer ordered the wrong SKU (due to a fault of his own);

  • If the customer does not like an SKU that is part of a bundle;

  • If the customer bought a phased-out SKU;

  • If the customer bought a “limited stock” SKU that is no longer available.
     

The correct process is: the original product needs to be returned and the customer refunded; and then the customer will have to place a new order in the webshop.

In addition to that, please also NEVER create Supplement Orders to unblock orders (any scenario that the order is not on the normal flow). 

The correct process is: to escalate via JIRA to the Order Management team or to the Back-Office team (if your market has a back-office team).

Instructions

From the customer's original order:

1. Click the kebab button to access the action bar.

2. Click on Create Supplement Order.

​

​

3. Select the Name of the product or SKU to be included in the supplement order and click on Next.

A supplement order page will pop up:

​

Search function

You can search by product Name or SKU.

When typing the SKU, the search function will display options after 4 characters are typed in. If you are pasting the entire SKU number, try deleting one character for the dropdown options to be shown.

​

 

4. Select the quantity of the SKU selected should be added to the supplement order.

5. Click Next.

​

​

6. Confirm order delivery details based on the original order (in case customers requests to change, follow customers' preference);

7. Click Next.

​

 

A summary of the supplement order will pop up.

8. Confirm customer email.

9. Select Supplement reason:

  • Replacement

  • Add-on

​

​

  • 10. Submit and the Supplement order will be created.

You can access the Supplement order by checking the original order "related page".
