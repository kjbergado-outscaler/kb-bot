Order Verification in Magento

Ensures that the order details match between Magento and Shopify before proceeding with any modifications.

  • Obtain the Shopify Order Number from the customer.

  • Open Magento and navigate to Sales → Orders.

​

​

When accessing orders in Magento, the interface presents a list of orders, which can be searched and sorted based on multiple information, such as order number, external order number, customer details, and shipping information.

​

​

  • Input the Shopify Order Number into the External Number field.

  • Verify the customer name, address, and order details.

    • If the order matches, proceed to the next steps.

    • If the order does not match, investigate by checking other reference details or contacting support.

​

Order Page Overview

​

​

​

​

 

  • Order Information: Here you can see the order number, purchase date, status and related orders.

  • Billing Address: This is an additional reference number that may sometimes be important.

  • Shipping Address: This is de delivery address and customer’s information.

  • Account Information: On the right side of the order screen, you can see account details such as the Store’s name and customer’s email address.

​

​

By clicking on the account name, you can access the account details and view related orders.

​

​

​

​

  • Comments: The comments section is used for tracking important the interactions. So, every time the retailer contacts us about any topics, we need to add a comment to the order. These comments can be highlighted and marked as important so all agents can see the note when they access the order.

​

​

If you click on the SKU, you’ll be able to see the product’s status in the “Log” field.

Processing a Refund

Customers may request a refund due to various reasons such as product unavailability, cancellations, delays or trial period returns.

Once the customer confirms they wish to proceed with a refund:

  • Confirm that the order status allows a refund.

  • Check if there are pending collections that need to be addressed first.

  • If no collections are pending and you are in a Cancellation stage:

    • Navigate to the Order Details page in Magento.

    • Click the Cancel Order button.

    • Select the cancellation reason and the Refund option.

    • Select Refund £XXX,XX or cancel payment.

    • Click Cancel entire order.

​

​

If you are in a Returns stage:

  • Click in Return Products;

​

​

  • Fill-out the information accordingly:

    • Return Products: select the returned product(s)

    • Quantity: add the quantity of returned SKUs

    • Refund to the customer: mark if the return should be followed by a refund. If it’s just a collection case, select “No”.

    • Warehouse Address: make sure to select the correct warehouse if the product is opened.

    • Reason: select if it’s a Delay, Product Claim, Logistic Claim, etc.

​

​

  • Click Save in the top-right corner.

The return will then be created and will appear in the Order page, as Product Returns.

​

​

  • Open JIRA and create a refund request ticket. Please refer to this article to check this process.

  • Ensure that the ticket number is added to Magento’s Order Notes.

  • Inform the customer about the refund timeline and expected processing time.

Order Cancellation

Customers may request to cancel an order before it is shipped.

Before you proceed, inform the customer that the order cannot be reinstated once canceled.

​

  • Confirm that the order is eligible for cancellation.

  • Navigate to Order Details in Magento.

  • Click Cancel Order.

​

​

  • Select Reason for Cancellation (e.g., out-of-stock, customer request).

  • Select the Refund £XXX,XX or cancel payment option.

  • Click Cancel entire order.

​

Compensation Requests

Customers may request compensation due to delays or product issues.

  • Try to deescalate the situation and explore non-monetary resolutions first.

  • If compensation is necessary:

    • Open Magento and navigate to the Discount section.

    • Calculate the eligible compensation percentage based on the Compensation Matrix.

    • Fill-out the mandatory information as if follows;

    • Apply the discount amount. Follow the format in the image below.

    • Click Save.

​

​

  • Submit a JIRA ticket for compensation approval.

    • Navigate to the AR D2C Portal > Refunds > Refund Requests.

    • Reason for Refund: choose the correspondent Compensation reason

    • Description: add the number of delayed days and attach a screenshot of the compensation matrix.

​

​

​

​

​

Make sure that the discount amount in Magento matches the JIRA ticket amount.

Managing Supplements (Missing Items, Exchanges, Order Adjustments)

Magento may not always reflect all items in an order due to Shopify integration issues. If an order is missing items or a customer requests an exchange, you need to create a supplement with price.

  • Identify missing or incorrect items by comparing Shopify and Magento orders.

  • Calculate the correct amount for missing items or exchange.

  • Create a Supplement Order in Magento with the correct details and the total price.

​

​

​

Removing Items

  • If there are any unwanted items in the supplement, select the item and click Remove. Then click Update Items to remove it.

Adding New Products

  • Search for Products: Click on Add Products and search for the product you need by SKU or name.

  • Select Product: Choose the appropriate product and select the quantity needed.

  • Add to Order: Once selected, click Add Product to Order to include it in the supplement order.

  • Verify Prices: Make sure to add the total amount paid by the customer.

​

​

​

​

​

​

Shipping Details

  • Confirm Shipping Address: Ensure the shipping address is correct. This information should never be changed unless there is a necessary adjustment (e.g., changing the recipient’s name/delivery address).

​

​

You should never change the billing address information.

​

  • Shipping Method: Select the Free Shipping option at the top. This is the default and should always be applied.

External ID (Optional)

  • External ID: In most cases, leave the External ID unchanged. However, the customer might provide you this information.

​

​

Order Information

  • Order History: Enter a description or reference (e.g. "Shopify integration error"), to clarify the reason for the supplement order.

  • Once all fields are completed, click Submit to create the supplement order.

Adding Comments

Follow the standard format when adding notes in Magento:

Nomenclature​

 

  • GDPR – Y or N​

  • TN – Case number on Sprinklr​

  • Q – Question​

  • ACT – Action​

 

Mandatory sentence: "This is a Store ticket. If you received it without being part of the Store team, please return it to the Store queue."​

 

Example​

In blue, the parts that are fixed, and, in orange, what the agents should add depending on the specific case. ​

 

GDPR: Y or N TN: #xxxxxxx. Q: WIMO. ACT: Apologized for the delay and explained delivery process. Additionally, also informed the customer about the stock issues we are having and that I will follow up tomorrow with them on the situation. This is a Store ticket. If you received it without being part of the Store team, please return it to the Store queue.

 

GDPR: Y/N (“Y” if authentication is done, “N” if it isn’t).
Ticket Number: Sprinklr’s case number
Customer Enquiry: Customer’s request
Action Taken: Our response/actions taken

Agent’s name: Auto-filled.

​

Marking the Comment as “Important”

  • Click on the drop-down menu beside the comment.

  • Select Important.

​

​

This way, the comment will be easily visible by all agents working on the order.

Escalations and Special Cases

Some orders may require manual intervention from the store or management, if it cannot be resolved through standard processes.

  • Identify if the issue requires store intervention (e.g., barcode errors, rejected orders).

  • If required, raise an escalation in the store’s communication channel.

  • If a barcode cancellation is needed:

    • Go to Catalog > Barcodes > Manage Barcodes.

    • Search for the order.

    • Change the status accordingly and Submit.
