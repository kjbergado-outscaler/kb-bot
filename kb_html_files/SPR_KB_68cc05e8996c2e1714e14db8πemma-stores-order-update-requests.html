Phased out SKUs

When we are phasing out one SKU code and starting to sell another, we sometimes have orders that are caught in the change and end up in the backlog. These SKU codes will need to be updated so that the order can be sent out.

​

In these cases, Order Management / Offline Customer Excellence will send CS the affected orders, along with the old SKU codes and the new SKU codes. CS will need to:

​

  1. Contact the customer to inform them of the change and check they are happy to continue with the updated version. Please let them know that their order might arrive in more than one delivery.

  2. Create a supplement with the new SKU codes.

  3. Cancel the old SKU codes from the original order.

Example:

Other Order Update requests include:

  • Orders returned to warehouse with no open return - CS needs to check with the customer and either create a supplement to resend, or return and refund the order.

  • Carrier is waiting for the customer to schedule delivery - CS needs to inform the customer of this.

  • Order has been delivered to a ParcelShop - CS needs to inform the customer of this.

  • Check with the Store Team if the order/products have been collected in-store.
