1. About the The Return Self-Service Portal

Note: In FR, UK, ES, PT, DE, NL, BE, DE, CH, AT from now on ALL requests for returns should go via the portal.

​

Why use the Return Self-Service Portal?

The return portal is an efficient and more customer friendly way to create a return for customers.

In this portal, the customers will be able to see all the items in their order and select the ones they want to return. There they will also indicate the reason, the quantity and the condition of the articles.

​

For returns only including accessories, all the requests will be fully automated. 
For mattresses and beds, the CS agents will still have to take action to approve the return after the deflection.

If the customer asks you to create the return, please first guide them through the process of using the return portal. You can help them through the process.

​

When doing this, please check if you can see the related Sprinklr ticket created via the return portal, so you can assist with handling the return. As the ticket may take some minutes to arrive, you can also go directly to ParcelLab portal and see all the requests there in real time.

What if the customer insists on creating the return via customer support? See the answer in the FAQ section below this article.

​

ATTENTION

The Return portal does not cover Magento orders. For those cases, when receiving a Sprinklr ticket that is related to an order from Magento, please handle it manually like the old process, and DO NOT re-direct customers to the portal.

How does the Return Portal work? 

Note: for the actual step-by-step article click here

i. For accessories (standalone items) 

For accessories (standalone items), customers can start the returns by themselves. Every request opened by the customer on the portal, will automatically create a return request and generate a label for the drop-off. 

​

In that scenario, no action will be required from the CS agents. The integration between Return Portal, FluentCommerce, and ParcelLab will handle automatically the returns. 

​

Recap: 

products considered as accessories for the return via drop-off: pillows, duvets, disassembled/self-assembly slatted frames, baby/kids mattresses, packed toppers or toppers up to 120cm.  


IMPORTANT - STANDALONE X NON-STANDALONE ITEMS: 

A standalone item is how we refer to a product that has only one SKU and also appears in OMS with only one SKU associated. 

The non-standalone items will be the ones that have only one SKU, however, due to system issues it will appear in FluentCommerce in the same way as a bundle or composite. But you check it, it is indeed only one item.

ii. For other products and non-standalone accessories

If the return request opened by the customer is for non-standalone accessories or other product categories (like mattresses or beds), the portal creates a Sprinklr ticket that requires that needs follow-up actions from CS/Sleep Experts agents. 

All the steps needed are explained below in this article. But it's basically: identify the reason for the return; try deflection; create a return, and inform the customer.

​

iii. For Bundles 

For most countries, the partial return of bundles is not allowed, with a few exceptions that are related to High-Level Escalations/Online Reputation cases. In those situations, agents need to contact supervisors first and they will escalate this to Performance Managers of the related country/market.

​

Additionally the French market also allow's partial returns.

3. FAQ

​

6. A) What if a customer reaches out to us via phone, chat or email?

If the request comes via phone or chat

  • For an accessory only >> redirect the customer to the portal in the webshop. If the customer prefers, you can help them by craeating the return for them via the portal on the webshop.

  • For only mattresses or beds >> agent handles it via normal return process

  • For returns including accessories AND mattress or bed >> agent handles it via normal return process 


If the request comes via email

  • For an accessory only  >> agent handles it through the portal in the webshop

  • For only mattresses or beds  >> agent handles it

  • For returns including accessories AND mattress or bed >> agent handles it via normal return process

​

6. B) What if the customer insists on doing the return via customer support instead of via the return portal?

If the customer still prefers your support to create the return, you can still create the return via the normal manual process, see description here. Before you do this, always check Fluent Commerce if no return has been created already, so we get double created returns.
