Scope

The aim of this SOP is to explain how to create a supplement in Magento.

This applies to all stores.

​

IMPORTANT: We have a limit of 3 supplements per order. ​

If, for some reason, the agents need to create a 4th supplement for the same order, they should escalate to their supervisors and the supervisor should reach out to the Order Management POC for review of the supplement need – Maigerlis Correia.

​

Instructions

In instances such as when the wrong item has been sent to a customer and it needs to be exchanged for something else or when an SKU has been changed, a supplement needs to be created.

​

Do NOT create supplement for order if the barcode status is:

  • WAITING FOR STOCK → SKU is OOS (no need create a supplement - the sku will continue to be "Waiting for Stock". In case stock is available but order remains "Waiting for Stock", contact to Order Management.

  • PACKING → Verify that the order is not a bundle order, as it may be awaiting another SKU whose status is "Waiting for Stock" - if it is a standard order, contact to Order Management.

  • REJECTED → The SKU has been rejected for the WH. In this case, contact to Order Management.

​

  • Select the barcode you wish to create a supplement for;

  • Go to Change Status → Deleted.

​

​

  • In Magento, open the order you want to supplement a barcode in.

  • Make a comment on why the supplement was created.

​

​

​

Important note for supplement orders where additional services are required:

​

These services need to be booked for the supplement order (please see information for each market).

Beds

  • For supplements for beds, if the supplment is created pre-deliver (e.g. a SKU change and not a replacement for a damaged component), please create a supplement for the full product.

  • Complete the requested data according to what has happened with the sku(s) and click Save.
