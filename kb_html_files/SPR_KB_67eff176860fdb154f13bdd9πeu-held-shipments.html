Held shipments

Always add at least 4 keywords (two words max to help identify the article).

Examples: replacement; supplement; wrong product; new product; product information

  • Definition

  • Scenario

  • Closing the loop

Definition

A shipment can be held by our delivery partners when a package or order is stalling at the carrier hub due to multiple reasons:

a. Missing details or wrong address

b. Customer refused the delivery

c. Customer was absent at delivery moment

d. Customer is not reachable

e. Shipment of multiple items and 1 or more boxes are missing

f. TBD

Scenarios

Wrong or incomplete address

The struggle to deliver comes from incorrect information or details on the customer address.

  1. Find the order on Fluent (if the carrier provides you with order number or fulfillment ref) or ParcelLab using the tracking code or by last name on Fluent.

  2. Check if there is any "visible" missing information on the address on the tab Attributes

  3. Check if customer has already contacted us; if Yes assign the case to yourself and use the canned response "XXXXXX" (go to step 5); if No, go to step 4.

  4. Contact customer (Outbound case) by using the canned response "XXXXXXXXX"

  5. Set the case in awaiting on response.

  6. Once the customer replies, we contact back the delivery partner to provide them with the missing details or the delivery specifics.

Missing part of the order

In some cases, the carrier can notify our Support that one or more of the packages in the order are missing.

Some carriers can identified what package is missing, other cannot.

Carrier inform us which box is missing:

4. Create a supplement for the missing SKU
