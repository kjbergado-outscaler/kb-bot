Introduction

In this article, you will find instructions on Case Status and which one to use after handling a customer request.

Case Status options

According to the customers' request and the follow-up actions, you need to inform the system of the outcome by inserting the correct status related to the case. In Sprinklr we have 5 cases statuses:

  • Awaiting Response

  • On Hold / Internal Hold

  • Pending Resolution Confirmation

  • Closed

  • No Response Required

  • Customer Closed

When to use each Case Status:

Case Status

Definition

Notes

Examples

Awaiting Response

To be used after the Agent replied to the customer, and now is awaiting response from customer in order to proceed.

For email cases: The customer will receive automatic reminder emails after 48 and after 96 hours. If there is no reply within 168 hours (7 days), the case will be auto-closed and a CSAT survey will be triggered.

In a Product Claim scenario, when you have asked customer to send picture; In any query when customer didn't provide enough nformation for you to find the order on the system.

On Hold / Internal Hold

On Hold: appears on the ACW window & Internal Hold appears when you check the status of the case. But they are the same thing.

To be used when the case resolution is pending on a 3rd party reply (ie. Partners - WHs/Carriers, Emma Internal - CE/OM/AR D2C, etc)

This status will occur when an Agent manually changes the status of a case as part of the process of escalating to 3rd parties or internal Emmies.

On this status, if the Customer replies anytime during the case being on “Hold”, the case will go back to the queue to be assigned.

In a Refusal upon Delivery scenario, when you have to open a refund request via JIRA ticket;

or when you need to edit an order and raised a ticket to AR D2C.

Pending Resolution Confirmation

Use this status if you consider the case as resolved for the customer, but an active confirmation of resolution from the customer is needed.

FOR EU MARKETS ONLY:

To be used when the Agent has resolved the customer query satisfactorily and the case is resolved.

​

If within 72h on this status the customer doesn't reply, the status will automatically be set as "Closed" and a CSAT Survey will be triggered.

FOR EU MARKETS ONLY:

For LiveChat, moving the case to "Pending Customer Resolution" will automacatilly trigger the CSAT Survey to the customer.

In a Return scenario, when you have done all actions needed and now have just informed customer about return date and refund timeline.

Closed

(This Status is no longer Available for EU Markets)

To be used when the Agent has resolved the customer query satisfactorily and the case is resolved.

Attention: The customer has to actively confirm that the matter is resolved for them.

  • For Email and Chat cases: Moving the case to the "Closed"-status will automatically trigger a CSAT Survey to the customer.

  • For Voice cases: In the ACW you only have the option to move the case status to ""Closed"" if you sent the customer to the Survey IVR before disconnecting the call. If you missed this, make sure to update the Customer profile in Sprinklr with the email address and move the status to "Pending Resolution Confirmation".

--> Bear in mind that if you did not solve the customers' issue and still move the case to "Closed", you might get a bad rating in the CSAT survey. So when in doubt, rather use "Pending Resolution Confirmation".

Agent has resolved the customer’s concern, confirming that the issue has been addressed and no further steps are required.

No Response Required

To be used when no reply is sent to the customer. It can only be triggerd by Macro (please check the Macro article).

--

For Spam/Unactionable cases and no answer to customer is needed.

Customer Closed

User actively closed the chat conversation on the website

This status is automatically applied to a chat case when the User actively closes the the chat conversation on the website.
