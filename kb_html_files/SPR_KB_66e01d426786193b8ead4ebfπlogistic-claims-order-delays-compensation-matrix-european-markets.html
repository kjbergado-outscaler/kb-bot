Introduction

In this article, you can find an explanation about what is the Compensation Matrix for all European markets and how to use it. The EU Matrix was created to set a standard and manage the compensation values that Emma offers to customers in specific situations.

Also, now we have 2 versions of the Compensation Matrix: one for regular cases that the CS agents handle; and the second one ONLY to be used by HLE/PRIO1 agents that handle the cases from the High-Level Escalations queues.

Additionally we also have the compensation feature for Delivery delay claims. (Live since April 24th)


ATTENTION: The compensation should be given per order (or item/SKU, if it's the case), but not per complaint. If there are multiple complaints, the most severe complaint should be chosen. The only exception is when there's a Logistic Claim and a Product Claim (check on the FAQs).


Before starting using the Compensation Matrix, it is necessary to understand which cases and scenarios are valid for a compensation to be offered to the customers.

Compensation Matrix Steps

Before starting, ensure these three Golden Rules:

1. Always check Fluent commerce comments before proceeding with a compensation.

2. It’s mandatory to always authenticate the customer before starting the compensation feature.

3. Make sure the order Reference matches the order where compensation is applicable

​

​

  1. Select Compensate button.

  2. Select Proceed.

  3. Select the eligible items that require compensation.

  4. Click Generate Preview.

  5. If everything is okay, click Compensate.

  6. Review and if accurate, click Finish.

  • Should we compensate the customer for a delay after or before they receive the order?
    Always when the customer receives the order. Please don't mention %s, just calculate the full value with the OD calculator (as we can do regular updates on the minimum and maximum values).

  • What's the step-by-step process for providing compensation for an order delay?
    When an order is delayed, we should inform the customer of the new Updated Delivery Date (if applicable). If there is no UDD information (invalid date), we should warn them that we don't expect the order to arrive in less than 8 weeks. Always let them know that we offer compensation after 14 days of delay. If they are willing to wait and be compensated, ask them to alert us as soon as they receive the order so we can calculate the compensation. Please avoid mentioning specific compensation rates.

​

  • Should we roundup the compensation values (ex. if it's 61.77€, offer 65€):
    Only cents, for example: if it's 35.62, offer 36€. Ideally we should always offer the exact values that appear in the OD calculator.

  • What to do if acustomer ordered a product with assembly included and the carrier didn't assemble during the delivery?
    The correct process is to contact the carrier in order to schedule a new assemble. If that happens, we don't offer compensation. But, if the customer contact us explaining that the carrier did not assemble the product but he did, please offer a maximum value of 25 euros in compensation.

  • Can I provide the maximum of compensation for Order Delays whenever I believe the customer should have it?
    No. For Order Delays, taking into account the compensation percentages in each bucket and the maximum values, the compensation will always be the lowest value between the two. Example: if the compensation value is 45€, but the maximum value for the bucket in the file is 50€, we will still pay 45€.

  • Can the customer cancel the order due to a delay?
    Yes, but if they have received compensation for a delay, this must be deducted from the full refund.

  • Should we offer compensation for the entire order or on a per-item basis if only one item (SKU) is delayed?

    If the situation is not with the entire order and we have the price split per item in OMS, we should be able to compensate on an item specific level. One example is a customer that bought a bundle with a bed, mattress and pillow. If there's a delay with only the pillow, the agent needs to specify the SKU that requires compensation in the Order 2 Cash Jira portal in order for us to compensate the customer just for the pillow's delay.

  • What to do when the order/item value is low and therefore offering the minimum compensation is still a lot?

    On W26, 2024 we decided to lower the minimum values to address this situation. An example is a $30 order where the reward for scenario 2 is 5% ($1.5). Considering that our minimum payout is $5, we should still pay $5.

  • Do we have to transfer the tickets to the HLE queue?
    No, you shouldn't need to transfer it. As stated, we should provide compensation ONCE the order is delivered to the customer. We understand that the prior process was to proactively compensate customers on some countries. For complaints that happen before June 5th, if repeating WIMO for +3 times (3 wrong updated delivery dates) in the same order, it's possible to transfer to the HLE queue. Before transferring it, please ensure that the item is not phased out (if applicable to your country) and write an internal note explaining why the ticket was escalated to the HLE queue.

  • What if the customer doesn't accept the compensation values?
    The compensation values are final and if the customer don't agree he can always cancel the order and get a full refund. The only exception is if it's a regular CS case but it's a legal threat. If this is the case, please escalate the situation to your TL and If he agrees, please open the ticket explaining that it's a legal threat, then it's possible to use the HLE Compensation Matrix instead of the regular EU CS one.

  • If we have initiated a compensation and the customer wants to cancel their order, do we refund them the full amount or do we deduct the compensation value from the full price?
    We will deduct the compensation value from the full price.

  • Will HLE initiate the compensation or will they reassign the ticket to our queue with an approval?
    They will initiate the refund.

  • If the customer was promised an incorrect amount by a previous agent according to the previous policy, what do we do?
    Send proof in Jira that the date of the promised compensation is before May 25th, 2024.

  • If a wrong product was shipped and there is a price difference that should be refunded to the customer, what should we do?
    If a customer places an order for $1,000 and mistakenly receives a similar product that costs $800 and wants to keep the product, we will refund the difference between the two ($200) plus the compensation for the $800 product. If the customer doesn't want to keep the product, they can return it as normal.
