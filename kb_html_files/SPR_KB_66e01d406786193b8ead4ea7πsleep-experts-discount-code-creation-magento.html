Introduction


In this article, you as a Sleep Experts agent will learn how to create use Magento to create voucher discounts for customers.



Sleep Experts codes


If a customer talked to the Sleep Experts team and was given a voucher/discount code to be used on the website, the discount code can be applied to the order as a discount. This should be done by the customer when placing the order on the website.


If the customer forgot to use the discount code, it can be applied as a normal discount -> forgot coupon code. The Sleep Experts agent needs to inform their supervisor who needs to send the coupon code to the Point Of Contact from Emma.


The Sleep Expert coupon code can be added after the purchase within the 100 nights trial period from the date the order was placed.

How to find a Sleep Expert coupon code?


The Sleep Expert codes always start with "SE". And you can search for it in Magento.

Click in "Promotions" -> "Coupon codes (read only)" -> type and search for it.



IMPORTANT: Freshdesk Ticketing Status information

Every time we face a customer decision about accepting or not the discount, ensure you leave the ticket as "Waiting on Customer..." and the ticket will be resolved automatically if the customer never gets back to us.



How to create a Sleep Experts Discount Code in Magento?



Creating a code:

You have to go to "Promotions" -> "Coupon codes By Agent)". Then it will open the creation code page, in which

you will be able to generate a voucher after selecting an option in every field. The fields are:


- “Shopping Cart Price rule” -> it is the rule the voucher applies on the order (for example 5% discount)
- “Contact source” -> the source through which our customers are contacting us

- “Market” -> the market from which our customers are contacting us
- “Outbound consent” -> Did the customer agree for us to keep his/her data and to be contacted by the agent after some time? If yes, through which source?
- “Reason” -> For which purpose did we give out the voucher?
- “New comment” -> This section should follow a very clear and precise rule that is “[customer contact] / [agreed call back date dd/mm/yy] / [personal comment]”

Checking all your created codes:


Once the coupon is created, the information characterizing it can’t be modified. Never try to delete a voucher. If you make one accidentally, or the details are wrong, just leave it and create a new one. - this won't negatively impact you or the customer.

After creating the voucher, it can be seen below where all the coupons belonging to the same agent are displayed.



Every field has a filter through which agents can be able to search and select different options to filter and display only certain coupons.

The filters have the following information:

- “Agent” -> Displays the agent name accordingly to the Magento user name.
- “Rule name” -> Rule characterizing the coupon.
- “Coupon code” -> Unique alfa-numeric code identifying every single voucher.
- “Coupon Created At” -> When the coupon is created. It is used to control the date when the coupon is given, meaning that every coupon should be created only when in contact to the customer. It is forbidden to create the coupon in advance and then hand it out to the customers.
- “Expiration date” -> When the coupon will expire.

- “Times used” -> When the coupon is not used the value is “0”. It will update once the coupon will be used to “1”. Remember that every coupon can be used only once.
- “Discount” -> Percentage value discounted by the order value.
- “Action” -> How the coupon applies its rule on the order.

- “Reason” -> It is the reason why we are giving out the coupon.

- “Comment” -> Displays the unique comment that has been previously written by the agent.
- “Contact Source” -> The source through which our customers are contacting us.

- “Order Store” -> The market from which our customers are contacting us.


Navigation tips:



The table is easily navigable through filters. It is designed to let the agent monitor their performance, looking at whether vouchers given were converted or not, filtering the coupons created to get back to the customer once it is not converted, and looking at the notes.

Through the filters on the top of the table, it will be possible to define some characteristics of the coupons created.

Here is some advice on how it could be used:

-> Through “Coupon Code Created At” it is possible to see all the coupons given out during a certain period to check how much of them were converted. Interesting insights can be gathered, understanding the most expensive orders that were converted during that period and looking at all the vouchers that were not used yet, getting back to customers that allowed us to.


-> Selecting “0” in “Time used”: By doing this all the vouchers that were not used can be seen making it easier to get back to customers that allow us to do so.


-> Using a repeatedly specific format in the “Comment” section can be helpful in grouping certain kinds of orders. For example, filtering the agreed call-back date can be used to call back some customers every day at the end of the day to increase customer service and conversion rate.
