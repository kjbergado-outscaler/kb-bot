Scope of application

This procedure applies to all orders managed in Magento where it is necessary to replace an SKU or resend a product for the following reasons:

  • Logistic claim:

    • Carrier or warehouse issue

    • Incorrect product sent (wrong SKU)

  • Product claim:

    • Manufacturing defect on the product

  • Requested SKU change:

    • Phase-out

    • Depletion

Waiting for delivery date

CS agent can modify the delivery date directly in Magento. Do not create a supplement.

Additionally, if the mattress cover is damaged, a supplement must NOT be created for the entire mattress. Instead, the cover should be sent separately as supplement.

The process to verify which cover corresponds to each mattress nomenclature is detailed in the following reference guide: Supplement for Covers (EU)

Also, NEVER create a supplement in the following situations:

  • If the customer does not like their SKU or wants to return it during the trial period (e.g., exchange product color).

  • If the customer ordered the wrong SKU (customer's own fault).

  • If the customer does not like an SKU that is part of a bundle.

  • If the customer purchased a ‚Äúlimited stock‚Äù SKU that is no longer available.

Correct Process for these cases:

  1. The original product must be returned.

  2. The customer should be refunded.

  3. The customer must place a new order in the store.

Furthermore, NEVER create Supplement Orders to unblock orders (any scenario where the order is not following the normal flow).

In caso the doubts contact with Order Management team.

üìùProcedure for Creating a Supplement

  1. Open the order in Magento.

  2. Leave a comment explaining the reason for creating the supplement.

3. Select the appropriate claim type:

  • Logistic Claim ‚Üí Issues with the carrier or warehouse.

  • Product Claim ‚Üí Issues with the product (manufacturing defect).

4. Complete the required information in the claim form and click Save.

5. In the Items Ordered section, remove the incorrect SKU(s):

  • Click Remove for each item.

  • Click Update Items and Qty's.

6. Click Add Products:

  • Search for the correct product to replace the previous one.

  • If it is a bundle SKU, select Bundle Product in the Type column.

  • Click Add Selected Products to Order.

7. Adjust the supplement item price:

  • The new product must have a value of 0‚Ç¨ (the customer has already paid for the original order).

  • No payment method should be selected.

8. Select Shipping Method:

  • Set it to Standard.

9. Ensure no email confirmation is sent to the customer.

10. Review all details and click Submit Order.

11. The system will create a supplement with the original order number followed by S1 (e.g., 123456-S1).

Exceptions

Any case not covered by this procedure must be consulted directly with the Order Management team for evaluation and personalized action.
