Return Management




Depending on certain outcomes of a request, you may need to trigger a return in the order management system or change its status.

e.g. Return (Trial period request), product, or a logistic claim where you may need to replace the product or refund the customer.



This article will guide you through the steps of triggering the reverse logistics in the Fluent Commerce system, which at the moment requires three main steps:







RETURN WITH REFUND x RETURN WITHOUT REFUND

Return with refund means returning the item and triggering a refund of the amount paid by the customer related to the items selected.


A Return without refund is a return that does not trigger the refund, for a scenario such as a replacement (supplement order to replace the original item returned).


Whenever you want to trigger reverse logistics to order, you will always have to select which one of the two options you would like to proceed with. The steps in the system will be always the same, the difference is only the option you select and the outcome of it (Should the customer be refunded?).

IMPORTANT - ONLY FOR DE, BE and NL


ONLY FOR 1MH returns (packed mattress and Unpacked toppers from 140cm wideness)


For Returns requests that are 1 Man-Handling, including the Pick-up Date is mandatory. That information must be there so 4PL can send it to the carrier, and without that, the return will not move forward.


Agents must ask customers for a date, which must be accommodated a minimum of 3 days ahead of the requested date. However, it's not guaranteed that the pick-up will happen on the day the customer has requested. The carrier will try from the date onwards, between 2-5 days afterward.


Example: The CS agent contacts the customer on the 10th of March to ask for a pick-up date, and the minimum timeframe accepted is for the pick-up date to be on the 13th of March.

How to Cancel a Return

You may face a situation in which is needed to cancel a return that was already created. For that, you can use the "Cancel" button inside the Return order created and select one of the reasons:

- Incorrect Return: Return created by mistake.
- Customer canceled return: The client decided to cancel the return and keep the product.
- Customer "No Response": Customer asked for the return, but then stopped replying to us and didn't confirm the return or the return arrangements.
- Return Expired: Only to be used by Emma internal.
- Customer "Not Encountered": The customer was not at home on the return date and also was not available to contact via phone.
- Out of carrier T&C: For when the cancellation happens because of a faulty carrier assignment (can be due to the weight, length, belt-size, unsuitable packaging, item condition, etc). For example, if an agent creates a return for a packed mattress, but when the carrier arrives, the item is unpacked. The carrier cannot collect and inform this reason. Then the return will be cancelled and another one will need to be opened to request the right carrier.
