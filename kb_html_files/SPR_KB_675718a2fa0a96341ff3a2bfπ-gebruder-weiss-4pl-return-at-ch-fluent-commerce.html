1. How to initiate a return with Gebrüder Weiss

Step 1: Ensure customer details: Phone number, email, and pick-up address must be correct. So the carrier can arrange the pick up.

Step 2: Select all the items, inform the quantities, product conditions (packed/unpacked) and return reason.

If the customer has to return a big item (mattress or bed) and an accessory, create only 1 return and add all the items. This way all items get picked up by the same carrier and the customer does not need to return any product separatly.

Step 3: Pick-up date: Leave empty.

Step 4: For Fluent Commerce only: Status will be 'Created' automatically. Do not mark the return as 'Organized' manually.

2. Important information for setting customer expectations

  • Gebrüder Weiss will contact the customer to schedule a pick-up.

  • Never promise a pick-up date; it must be arranged with the carrier.

  • Gebrüder Weiss will send one or two employees with a truck for collection.

3. How to verify if the return was created correctly

In Fluent Commerce

  • The customer contact details are correct

  • The return status must be set to 'Created'

  • The return date must be left empty

    (After saving, this might be updated to 'GW_AT' or 'GW_DE' automatically by the tool, this is not an issue)

  • The carrier field is left empty

Note: Do not manually set the status to 'Organized'. If the return status is set to 'organized' manually, the return will not be recognized by our platforms, and it will never be forwarded to the return carrier. Please ensure the return status is 'Created' after you created the return.

​

4. How are status updates handled for the return?

  • 4PL CS updates the status in the Order Management System (OMS) within 7-10 days.

  • For detailed updates, check 4PL's Optilo platform.

  • If no updates after 10 days, escalate via the 4PL CS Jira portal .

5. How the refund is triggered

Fluent Commerce:

Status updates to 'Complete' automatically trigger the refund.

Note: Banks or payment service providers (PSP) will have different deadlines to have the amount available reflected in their account. More detailed timelines per PSP can be found in the article 'Refund request'.

If the customer reaches us after we already completed the refund, they should reach out directly to their bank or payment provider company to get further information and more specific deadlines.

6. When can this return ticket be considered 'Resolved'?

Once you have successfully created the return and have verified that it was created correctly, you can mark the case as 'resolved'.
