Potential Challenges

We create manual returns in WARNING due to issues that can occur when processing returns automatically in Fluent Commerce. This ensures the process is completed smoothly and accurately.

To streamline this process, follow the instructions below.

Instructions

  1. Access the WARNING Portal: Log in to your WARNING account here.

  2. Initiate the Return Process: Navigate to the designated return function within the system's menu. This is typically located as the second option from the left.

  3. Complete Return Details: Fill in all required fields pertaining to the return, such as order number, SKUs and customer's personal information. This will advance you to the next stage.

Change the field on the far right from"Livraison" (Delivery) to "Reprise" (Return).

​

  • Proceed to fill all the mandatory fields (marked in red asteriks and highlighted in yellow).

​

E.G of the filled-out information.

​

  • Click the "Next" button to continue the return processing workflow.

  • Once all the required fields are finished, validate the information by ticking the icon (marked with a red arrow).

  • Specify Return Items: Enter the SKUs of all products being returned.

  • The next step would be to fill in the refrences SKUs, quantity and weight.

Note: If the return contains more than one item you can click on the plus sign (signaled with the yellow arrow) to add the remaining items.

  • Review: Carefully review all entered data for accuracy.

​ Important: Make sure the correct data was added before confirming the return. Entering the wrong information can hinder the return process and cause significant delays.

​

  • Save: After carefully reviewing, click on the icon "Créer le récépissé" (Create Receipt) marked by the red arrow.

Potential Challenges

  • Incomplete Customer Information: Missing crucial data points, such as the customer's phone number, can significantly impede the return processing timeline.

  • System Interruptions: Unexpected logouts from the WARNING portal can disrupt the workflow and necessitate re-entering information, leading to potential delays.

    ​

By following these steps and proactively addressing potential challenges, you will be able to effectively process manual returns within the WARNING portal.
