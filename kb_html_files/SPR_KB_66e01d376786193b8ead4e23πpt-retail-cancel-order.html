Cancelation Request

Magento

Cancellation in the Order Management System is only possible up to the order is submitted to WH. The cancellation button will be unavailable if not possible anymore.

​

Until Packing Status

If the request to cancel the order arrives with the status still in "Packing", it means the order was not yet sent to the WH. We can cancel the order and it will not pass to the WH.

​

Fluent Commerce

Cancellation in the Order Management System is only possible until the Assigned . The cancellation button will be available but its not possible to cancel the order.

​

Up to Assigned Status

If the request to cancel the order arrives with the status still in "Assigned", it means the order was not yet sent to the WH. We can cancel the order and it will not pass to the WH.

​

Instructions: Click on the 3 dots in the top right corner, select “Cancel order items,” choose the items, and provide the reason. A credit note will not created, as it has not been invoiced yet (it is only invoiced when delivered).

You can find details on how to proceed step by step with a cancellation under the order management system (OMS) related article.

After Packing/Assigned Status

If the request to cancel the orders arrives after the "Packing/assigned" status, it means that the order was already sent to the WH. In this case, we'll need to contact the WH or Carrier (depending where the order is) so they can update the delivery information on their hand.

  • Order with WH: If the order is still in the WH, please open a jira to the WH (Offline WH Communication 2 - Jira Service Management (atlassian.net) ) and ask to cancel the Order Number and Barcodes that you want to see canceled. If the cancellation is confirmed, inform the retailer. No credit note is needed because it has not been invoiced yet. If cancellation is not possible, then it is treated as dispatched.

  • Order with Carrier:If the order is already with the Carrier, please open a child ticket to the respective carrier (or carriers, in case it's being handled by more than one carrier) and ask to cancel the order and return the material to the WH.

​

  • Dispatched: Inform that it was not possible to cancel, and tell the customer to refuse the delivery. The product will be returned to us.

    When the product returns to us, the status in Fluent should change to “return_sender_complete,” then we process the return with refund in Fluent. This doesn't happen often. This is why we ask the customer to inform us that they refused the delivery, and we can process the return with refund. The tracking link can also confirm that the order was rejected, as the carrier updates the status.

    ​

    If the customer accepts the delivery, then we organize the return with refund for the customer. For this, we create the return in the system and organize it with the carrier. Once the product is back with us, we go to the return created in the system, mark it as complete, and the refund initiates automatically.

Credit Notes for Canceled Orders

If the delivery has not been made and the order is canceled, no invoice has been issued, so a credit note is not needed. We can reassure the customer and inform them that the order will not be invoiced.

​

Returned Order

If we need to cancel an order that we see has been returned to us (customer absent or any other reason), we must check if it is “complete” in the system. If it is complete, it does not need to be canceled.

Refund the Retailer

If an invoice was created for the order, we'll need to refund the retailer by issuing a Credit Memo.

Magento

Every time you need to create a Credit Memo, open the Retail Backoffice Portal and choose Returns -> Create Credit Memos

​

Please give the following information to the Backoffice team:

  • Order Number

  • Amount / SKU to refund

Fluent Commerce

In Fluent Commerce we need to complete the return, after completing the return the agent must wait until 48 hours for the CM to be available in WINSCP (and the agent can download and sent to the retailer, if after this time its not updated there, please open a jira to AR.

​

After the creation of the Credit Memo, kindly inform the retailer that the document has been generated. Don't send this document unless the retailer asks for it.

Exception: For Miguel Barros (gerente@colchoesecompanhia.pt) always send the Credit Memos when generated.

Customer Inquiries & Sprinklr Ticket Type

After the order is already placed, customers may reach us to cancel their order.

Customer Inquiries for canceling the order can be listed as:

  1. Cancel order - personal reason

  2. Cancel order - delay

  3. Cancel order - Wrong product/ size/SKU

  4. Cancel order - duplicated order


Ensure you always provide the related Ticket type & subtype based on the customer request.
