How to make an order?

1. Fill out order file

We recommend using the file provided below. It is an example Excel file with some validation you can easily use to create your own order file.

​

xlsx

Manual order xlsx template - 20250404.xlsx
0.02 MB

​

Each line in the file represents one line item of an order. Therefore following fields are mandatory for each line:

​

field

description

order_group_id

This field allows to group line items to one order. Needs to be the same for each line and unique for each order in the same file. It enables to have multiple orders in one file with multiple line items. We suggest to just work with increasing values like 1, 2, 3, …

sku

The SKU of the ordered line item. It can be an Emma SKU, the retailer SKU or an EAN.

qty

The quantity of the ordered line item.

delivery_date

Delivery date for the specific line item.

​

All other information of an order should be set once per order in the same line, we recommend using the first line of the order group. The following fields are mandatory for each group.

​

field

description

emma_customer_id aka BU

key

The unique identifier of the customer (e.g. BU-XXXXX).

external_order_id

The order id of our customer.

order_type

Needs to be either wholesale or wholesale end customer. It defines if an order is sent to our customer or to the end customer directly.

shipping_method

Needs to be either standard or planned. Always use standard unless the delivery needs an appointment.

firstname_shipping

First name of the shipping address. Can be empty if company_shipping is set.

lastname_shipping

Last name of the shipping address. Can be empty if company_shipping is set.

company_shipping

Company name of the shipping address. Length to max 45 characters.

Can be empty if firstname_shipping and lastname_shipping are set.

street_shipping

Street of the shipping address.

postcode_shipping

Postcode of the shipping address.

city_shipping

City of the shipping address.

country_shipping

Country of the shipping address as country code (e.g. FR, DE, UK,..). You can find all possible countries within Settings > International in Commerctools. Nevertheless you can only use the country the customer is operating in.

​

​

All other fields are optional:

​

field

description

region_shipping

Region of the shipping address.

email_shipping

E-mail address of the shipping address.

telephone_shipping

Telephone number of the shipping address.

discount_code

Discount code to apply a discount on the order and/or order line items.

 

2. Upload file

As last step you need to upload the file in Wholesale Portal » Order Input.

​

​

3. Check Output of file processing

Once the file starts being processed, you can see the status of it and how many orders are still to be processed. You’re immediately see in the page 2 tabs:

  • Errors found

    • a table is displayed with the External Order ID, the BU key and the error details

    • this means that the order was not created and action needs to be taken, according to the displayed error message

    • once you take action on the errors, you can resubmit a file with the orders that failed

    • ATTENTION: make sure to NOT update the exact same file if it includes orders that were successfully created at the first attempt, or those orders will be duplicated.

Next to the tab name there’s a number indicating how many orders are there in each case.

​

​

  • Orders created

    • a table with the columns Order ID, BU Key and External ID is created

    • on mouse hover on each of these fields you can see the ‘copy to clipboard’ icon, just click on it if you need to copy that information

    • Order ID has direct link to the order in Fluent Commerce

 

Examples of error messages:

​

Error name

Message

Required action

Country prefix not found error

Prefix for country code xxx not found.

Make sure you only use countries we are currently supporting. Also make sure you used the abbreviation of the country.

Incomplete shipping address error

Aggregated orders advise group has incomplete address.

Make sure you have filled all mandatory shipping address fields in the file.

Invalid data error

Error occurred for field xxx with value xxx. Expected value is xxx.

Make sure the field is correctly set in the file you uploaded.

Duplicate sku error

… Multiple lines for the SKU xxx in the order.

After replacing the retailer sku with the Emma internal sku, there were two line items with the same sku in the order. Ensure each sku is only present once.

Variants not found for skus error

… SKUs xxx, yyy not found.

The variant skus in the order are not part of that BU portfolio.

Missing part skus error

… Composite product variant with SKU xxx has component yyy with no part SKUs.

The component of the composite product (e.g. headboard) requires value(s) in the part SKUs field in Commercetools.
