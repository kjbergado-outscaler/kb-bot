NOTE: Use this SOP only to report issues that deal with warehousing (stock issues, wrong orders sent, transmission issues, warehouse specific escalations) or last mile and returns issues (no orders updates from carriers, tracking link issues, complaints, carrier specific escalations)

Link to the Issue Reporting File: Issue Reporting- MasterFile.xlsx

PLEASE NOTE:
- The Issues Listed Here Will Be Checked Once A Week On Fridays
- Kindly Use Your Best Discretion To Only Report Issues If They Are Relevant For The Last Mile Manager To Intervene And After You Have Already Tried To Solve The Issue On Your End And It Has Not Helped.

  • Please categorize your issues correctly as:
    WH- Warehousing (related to the DB Schenker UK- Tamworth)
    LM- Last Mile (related to UPS, DPD or Rhenus)
    RL- Reverse Logistics (related to returns collection requests)

  • Once you add the issue, please change the status to “OPEN”.

  • Once The Logistics Manager has picked up the issue with the related stakeholders, they will change it to “IN PROGRESS- LM”. Please do not use this status.

  • If there is an issues that no longer needs attention, please mark it as “CANCELLED” instead of deleting it from the file.

  • If more documentation/ information is needed from the customer/ retailer to action an issue, it will be marked as “PENDING CUSTOMER INFO”

  • If you or the customer needs any kind of information from the Logistics Manager, please mark it as “PEDING LM INFO”

  • When it is finally complete, it will all be marked as “CLOSED”.

    NOTE: Please ensure to always keep issues in the right statuses to keep proper traceability and to keep the list clean.
