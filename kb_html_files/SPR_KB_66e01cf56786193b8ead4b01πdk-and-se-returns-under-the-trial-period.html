Returns regarding the 100-day trial period in Denmark and Sweden


Emma’s offer of the 100-day trial period remains valid. That means we will follow the normal process and must verify if the return was created within 100 days after the last item was delivered to the customer. For the full process, look here.

In Denmark and Sweden, we offer 100-day trial on all product categories

If the trial period has expired, the process stops here.


For the full flow of the return process, see the image below. Below the image, the steps are described in further detail.



Step 1: Perform a return via a donation to a charity

  1. Create the return in Fluent as you would do normally.
    Set the Carrier field to 'Charity'
  2. Request the pick-up from the charity & inform the customer:
    1. Denmark:
      1. Use this file to allocate a charity nearby the customer
      2. If there is a contact person for the postcode, write an e-mail to the contact person with the customers contact info (do not put the customer or Bent Ole in CC):
        1. If pick-up is possible within 7 days (5 working days),contact the customer and schedule a pick-up (sometimes also drop-off is possible, but this must be aligned between the charity and the customer) -> Set the status in FluentCommerce to ORGANIZED when the pick-up request is confirmed
        2. If pick-up request was confirmed, but no one is picking the return up as agreed, please contact the contact person again and include Bent Ole (bop@dca.dk) in CC for further investigation.
      3. If there is no contact person for the postcode (no entry in the excel), please contact Bent Ole (bop@dca.dk) to check if he can find a different option in this area
        Cannot get a hold of Bent Ole? Please try
        1. Paul: pakl@dca.dk, or
        2. Lars: lmag@dca.dk
      4. Contact person will send an e-mail when pick-up was done successfully then you can confirm the return as RECIEVED in Fluentcommerce
      5. Contact person will send an e-mail if pick-up was cancelled by customer (e.g. refused or several unsuccessful pick-up attempts)
        1. If the contact person reports that the customer already donated it in another way in the meantime, please follow up with the customer to receive a proof for this (picture or receipt)
      6. Once you receive the 'proof of return / pick up / donation' from the customer, you can mark the returns as COMPLETE in Fluent. This will automatically start the refund process.

  3. Inform the customer:
    • Sweden:
      1. Ask the customer to contact a free-of-choice charity & donate the return to them
      2. If the customer does not have any preferences, check this file for a charity nearby the customers address and recommend them to the customer
      3. If there is no charity in the excel file that’s nearby the customer, google a close charity and recommend this one to the customer
      4. If you cannot find a charity nearby via google, the customer may donate it to someone in need (e.g. a shelter of an elderly person)
    • Ask the customer to provide a picture of the pick-up of the return and then confirm the return as received in Fluent Commerce (if the customer cannot provide pictures, worst case alternative is a handwritten note confirming the receipt, signed by the recipient of the donation)

  4. How to handle return issues:
    1. For any reoccurring issue please align with your supervisor.


Step 2: If no charity is available (within 7 days), check product category and create return via DSV

The return process is different for different product categories. For accessories, move to step 2A. For packed mattresses and beds, move to step 2B. For unpacked mattresses and beds, move to step for 2C.

If, for any reason, DSV cannot arrange the pick-up, you will need to try and arrange pick-up by a charity. For this, see step 5.


Step 2A: Creating returns for accessories

Returns with DSV need to be created both in Fluent Commerce, and then separately triggered with DSV. We communicate with DSV via the DSV x Emma Jira Portal. The detailed steps are outlined below:

Step 1: Initiate the Return Request in Order Management System (OMS)

  • Platform Access: Access Magento or Fluent Commerce to initiate a return request.
  • Manual Submission: Fill out the necessary details manually to begin the return process.

Step 2: Submit Return Details in DSV x Emma Jira Portal

  • Navigate to the Jira Portal and select ‘Pickup request’.
  • Carefully fill in all details regarding the return on the Jira Portal and submit.


Step 3: Wait for confirmation from DSV

  • Processing Time: Allow time for DSV to review and confirm the return request manually.
  • Communication: Stay updated via email or platform notifications regarding the status of your return request.


Step 4: Once you received the confirmation + drop off label from DSV, share it with the customer

  • The customer will need to drop off the item, using the drop off label
  • It is important that the customer keeps the proof of return! We need it to trigger the refund process.


Step 5: Use the Proof of Return to trigger the refund

  • When we have the proof of return, complete the return in OMS. This will automatically trigger the refund.

Step 2B: Creating returns for packed mattresses and beds

Returns with DSV need to be created both in Fluent Commerce, and then separately triggered with DSV. We communicate with DSV via the DSV x Emma Jira Portal. The detailed steps are outlined below:

Step 1: Initiate the Return Request in Order Management System (OMS)

  • Platform Access: Access Magento or Fluent Commerce to initiate a return request.
  • Manual Submission: Fill out the necessary details manually to begin the return process.

Step 2: Submit Return Details in DSV x Emma Jira Portal

  • Navigate to the Jira Portal, and select ‘Pickup request’.
  • Carefully fill in all details regarding the return on the Jira Portal, and submit.


Step 3: Await Confirmation

  • Processing Time: Allow time for DSV to review and confirm the return request manually.
  • Communication: Stay updated via email or platform notifications regarding the status of your return request.

Step 4: Prepare the customer for Pickup

  • Packaging: Inform the customer to ensure the product is securely packed and ready for pickup.
  • Labelling: Attach any necessary labels or documentation provided by DSV for clear identification.
  • Scheduled Pickup: Coordinate with DSV for a convenient pickup time and location.
  • Accessibility: Ensure the product is easily accessible for DSV’s logistics team during pickup.

Step 6: Confirmation of Receipt

  • Tracking: Monitor the return process through DSV’s tracking system.
  • Receipt Confirmation: Receive confirmation once DSV has successfully received the returned product.

Step 7: Refund will be triggered automatically you mark the return as Complete in the OMS

  • Inspection: DSV will inspect the returned product to verify its condition.
  • Status Update: Updates on the return status need to be provided manually through Magento or Fluent Commerce platforms.

  • Use the Proof of Return to trigger the refund: When you have the proof of return, complete the return in OMS. This will automatically trigger the refund.


Step 3: Offer a full refund without return

Only in case no charity was available, and the DSV warehouse has been closed, we will offer the customer a refund without performing a return. For DK and PL, we will inform customers that we will refund them, and they can keep/discard/donate the product. We do this because it is simpler and more cost efficient than creating a return.

You can share the below message with the customer: 

"We are writing to inform you that we will be issuing a refund for your recent purchase of [Product Name]. 

As an exception, there is no need to return the product to us. You are welcome to keep, discard, or donate the item as you see fit.

If you wish to confirm this refund and proceed with the process, please reply to this email at your earliest convenience. Our team will ensure the refund is processed promptly.

Thank you for your continued support and understanding."



If the customer accepts, move to step 3. If the customer still insists on having a pick-up service, we (Emma) need to provide that service. Please then move to step 4.

Step 3A: Process a refund without a return

  1. In Fluent Commerce, create a RETURN with REFUND.
  2. As the Carrier, add "Customer kept the product" and move it to ORGANIZED.
  3. Ensure you have written confirmation from the customer that they accept the refund, before you proceed.
  4. Afterwards, move the status of the return to COMPLETE (see image below). This will automatically trigger the full refund of the item. On the comment field in Fluent, write down "Customer kept the product".
    Remember, if you do not see the ‘Mark complete’ option in Fluent Commerce, make sure to scroll to the far-right side of the page, or zoom out until the options becomes visible.
  5. Remember to inform the customer that it can take ~14 days for the refund to be processed.


Figure 1: How to mark a return as 'COMPLETE' in Fluent Commerce

After this, the return process is completed.

FAQ

  • If there are any issues with the return, what do we do?

Use the appropriate field on the DSV x Jira Portal to get in contact with someone from DSV to answer your question.


The options you have, are:

  1. Missing tracking update
  2. Proof of delivery request
  3. Missed delivery
  4. Missed pick-up
  5. Address change

  • What does the charity do with the products?

For Denmark, our partner FKN, sells the goods in their recycling and Wefood stores.
