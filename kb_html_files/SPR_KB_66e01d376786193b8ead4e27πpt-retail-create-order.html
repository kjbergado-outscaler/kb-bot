Retail Orders

Regular Flow

The regular orders in retail are placed via 3 different ways:

  • Retailer Portal: Available via this link by scrolling until the end of the website page. Here you can find a Manual that explains how to use the Retailer Portal; you can also share this presentation with the retailers in case they have any doubt.

All Traditional Retailers

  • EDI: Some retailers have system integration with our Order Management System. This means that the retailers place the order in their OMS and it's automatically transferred to our OMS.

Jysk

  • FTP: A similar but simpler integration than EDI. The orders enter directly in Emma's OMS.

La Redoute

  • Backoffice: For retailers who we don't have an EDI developed yet, and don't have conditions to create the orders on Retailer Portal. Retailers send us the orders via a specific platform and our Backoffice team is responsible to create them in our OMS.

Whenever the client has problems with the retails portal, we should ask for pictures and escalate to Isabel.

Exceptions

Regular orders are created via Retailer Portal or EDI, but there are still some exceptions:

  • Auchan Orders: Since we don't have an EDI integrated with Auchan OMS System, Auchan Orders arrive to the CS and need to be created manually by the backoffice.

Find more about Auchan Orders in this article.

  • WH Orders: WH deliveries might arrive through the CS and must be escalated to Backoffice for order creation.

Find more about Sonae in this article.

  • Pallet Orders: Orders with big quantities delivered in the retailer WH, usually with a discount price.

Find more about Volume Orders in this article.

  • POS Orders: Exposition orders so retailers can have the products in store. Most of these are free of charge for the Retailer.

Find more about POS Orders in this article.

  • B2B Partners (One-Time Retailers): One-time Retailers orders are pontual orders made by companies that don't have a partnership with us, so that can they benefit of 0% VAT. These orders need to be created by the backoffice and arrive to the CS through the Sales Team.

Find more about One-Time Retailers in this article.

These exceptional orders arrive to our CS and have to be sent to the Backoffice for order creation in Emma's OMS.

Order Creation

Whenever you receive a request to create any of these orders, please forward your request to the Retail Backoffice.

Find more about the Retail Backoffice in this article.

Truck Orders x Pallet Orders

Open the Retail Backoffice Portal and choose Pallets Order Creation. Never use Truck Order.

POS Orders

Open the Retail Backoffice Portal and choose Order Creation -> POS Order Creation

Please give the following information to the Backoffice team:

  • BU

  • External Number

  • Delivery Address

  • SKUs

After creating the request, paste the Jira number as an internal comment in the Sprinklr case.

Once the Jira is completed, confirm that the order is correct and send the Order Number and Expected Delivery Time to the retailer.

One-Time Retailers - B2B PARTNER

Please proceed with the order only if payment has already been received.

You will receive this order request via Sales team (Bernardo Sequeiro, Joana Peixe, or Gonçalo Lufinha)They will send the payment proof and ask us to place the order. If you receive an email from someone from another team without payment proof, you should escalate to Isabel to ask if we can proceed with the orders. After receiving confirmation, we can then create the order via JIRA to the BO team.

​

Before inciating the request of the order, please forward the payment proof to receivables.pt@emma-sleep.com with Sara Biscaia in CC sara.biscaia@emma-sleep.com.

After this, open the Retail Backoffice Portal and choose Order Creation -> Single Order Creation

Please give the following information to the Backoffice team:

  • BU (If it's the first purchase and there's no BU, please open a Jira to AR and request the BU)

  • Client's name

  • Billing Address

  • Client’s E-mail

  • Delivery Address

  • SKUs

After creating the request, paste the Jira number as an internal comment in the Sprinklr case.

Once the Jira is completed, confirm that the order is correct and send the Expected Delivery Time and Order Confirmation document to the one-time retailer. If any of the products appear as Out Of Stock, please inform that these SKUs may arrive later due to lack of stock.

Commercial Requests

When receiving tickets from the commercial team requesting us to create an order (it could be a request from Cláudia or Bernardo), please ensure to CC the customer's email, which is typically included in the correspondence they have shared with us.

BU Key

The BU Key is the retailer’s code in FluentCommerce, and we can find it in the file PT_PartnerStore_Accounts.xlsx (sharepoint.com).

BU Key Creation

When we need to create an order for a B2B Partner retailer (they only buy occasionally), we first need to create the account in our system (BU Key). For this, we open a JIRA ticket to the AR team and click on the option: “Customer creation in Commerce Tools.” You can use this as the description in JIRA: Good morning, The customer wants to place a new order. This customer is a One-Time Retailer and has already made the payment. We therefore need the BU to create the order.

Complete Process:

  1. Request the creation of BU with a JIRA ticket to AR.

  2. After the BU is created, open a JIRA ticket to BO to request the order creation.

  3. Confirm the order with the retailer.

  4. Forward to remittance@bettzeit.com with the payment proof and the order number.

  5. Close the ticket: add here the new BU key Portugal - B2B Partners.xlsx (sharepoint.com)
