Additional expectations/best practices 

  • Work based on SLA’s 

  • FRT: 8HR; RT: 24HR; CPH: >4 

  • If there are cases breaching/that have breached FRT/RT – they are priority for assignment 

  • Agents focusing on manually assigned cases are expected to exceed the CPH target via quicker/bulk actions. 

  • Cross-check data sources for swift action 

  • Compare the number of total cases “new” and “awaiting assignment” vs. the backlog outputs within the Supervisor Dashboard. If there is mismatching data, deep dive and escalate to BPO Management with adequate insights. 

  • Case status process compliance 

  • Cases that are escalated internally shall only be set as “Internal Hold” for adequate follow-up 

  • As per temporary UK guidelines, “On Hold” should not be used 

  • Supervisors should use raw data from Supervisor dashboard to manually revert “on hold” cases to “internal hold” if applicable. 

​

Daily Follow-ups 

The Daily Report should be shared via email by 12PM daily, reflecting the output of the previous business day. 

It should be based on available Sprinklr reports, and leveraged by Supervisor(s)/Operations Managements to highlight the following items: 

  • Team impact – productivity; engagement dashboard actions; etc. 

  • Team effort – hours worked per role 

  • Mismatching data (followed by data-driven insights) 

  • Actions taken or recommended (if non-BPO-controllable) 

  • Technical CRM escalations (followed by data-driven insights) 

  • Help needed items
