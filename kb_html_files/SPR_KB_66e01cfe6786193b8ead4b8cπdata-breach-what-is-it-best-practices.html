Introduction

At Emma, the security of customer data and sensitive information is of utmost importance. As our customer support, you play a crucial role in identifying and reporting potential security incidents, data breaches, and phishing emails. This article explains what is a data breach and best practices to avoid it from happening.


What is a data breach?

This is not the same as a DSR or a GDPR request. A data breach is an incident where confidential, sensitive, or protected information is accessed, disclosed, or used by an unauthorized person or entity. This can happen due to various reasons such as human error, cyber-attacks, or system vulnerabilities.


Best practices to avoid data breaches and data security issues

  • Verify Customer Identity: Always follow Emma’s guidelines to verify the customer’s identity.

  • Data Minimization: Only collect and use the data necessary to fulfill the customer’s request.

  • No CC or BCC: Agents are not allowed to use the CC (carbon copy) or BCC (blind carbon copy) fields when sending emails to customers. All communications should be directed to a single recipient at a time.

  • Confidentiality: Treat all customer information as confidential. Do not disclose any customer information to unauthorized parties (REMEMBER: family and friends are unauthorized parties unless the customer provides consent to share the data with them!).

  • Accurate Information: Double-check all information before sending to ensure that it pertains to the correct customer.

  • Attachments: Ensure any attachments are meant for the intended recipient and are free from viruses or malware.

  • Order Confirmation: Verify the order details and customer information before sending a confirmation email.

Examples of data breaches and prevention measures

  1. Misaddressed Email: An agent sends an order confirmation to the wrong customer, revealing order details and personal information.
    Prevention: Verify the recipient’s email address and order details before sending. Implement a system of checks to catch mismatches.

  2. Incorrect Attachment: An agent attaches a file containing another customer’s order history to an email. Prevention: Verify attachments before sending. Ensure the file name and contents match the intended customer’s information.

  3. Unauthorized Access by Former Employees: A former employee still has access to the email system and customer information, which they use maliciously.
    Prevention: Immediately revoke access to all systems and data upon employee termination. Regularly audit access logs to ensure only authorized personnel have access.

  4. Data Entry Error: An agent mistakenly enters another customer’s information into a customer’s account, leading to a mix-up of orders and personal details.

    Prevention: Implement double-checking procedures for data entry.
